### Gap I1 — Phase 3 overlays cleared by Phase 6 Measure loop

**File:** `WPF_Example/Custom/Sequence/Inspection/Action_FAIMeasurement.cs:190`

`EStep.Measure` unconditionally assigns `pMyContext.InspectionOverlays = new List<EdgeInspectionOverlay>()` after the measurement loop. `EdgePairDistanceMeasurement.TryExecute` never writes to `ActionContext.InspectionOverlays`. `HalconDisplayService` therefore receives an empty overlay list — the green/red edge lines and cyan DistLine from Phase 3 (Plan 03-02 Task 1) no longer render.

**Affected requirements:** ALG-01, ALG-02, ALG-04 (visualization component).
**Severity:** blocker for visualization; measurements still flow to DataGrid so OK/NG judgement is intact.
**Fix direction:** Extend `MeasurementBase.TryExecute` signature (or add an overlay collector parameter) so each Measurement contributes `EdgeInspectionOverlay`s. Accumulate into `pMyContext.InspectionOverlays` inside the loop instead of replacing it.
