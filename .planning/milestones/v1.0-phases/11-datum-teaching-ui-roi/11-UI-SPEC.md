# Phase 11 — UI Design Contract: datum-teaching-ui-roi

> Visual and interaction contract for Phase 11 (WR-RT-01/03/04 resolution: Circle ROI drawing + Datum teaching UI + workflow guidance).
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.

## Layout Contract

### Canvas toolbar (extended)

Existing MainView.xaml:62-111 canvas toolbar strip stays at `Height=36` on Grid.Row=0. Phase 11 adds 3 new controls **inside the existing left-aligned StackPanel** (MainView.xaml:69) and reuses the existing `RoiToggleButtonStyle` resource:

```
[canvasToolbar strip — Height=36, Background=#111827, Padding=8,4]

  [Rect ROI]  [Polygon ROI]  [Circle ROI] | [Teach Datum] | [Calibrate] [Test FAI]
  ^^ existing ^^              ^^ NEW ^^     ^^ NEW ^^       ^^existing^^  ^^NEW^^
                                                                          
       ──── label_drawHint (centered, info/success/error) ────       label_pointCount (right)
```

Grouping rule: The three ROI-shape toggles (Rect / Polygon / Circle) form group 1. Teach Datum sits alone in group 2 (conceptually distinct — not an ROI shape, a workflow mode). Calibrate + Test FAI form group 3 (action buttons, not toggles). Groups are separated by `<Separator Margin="4,4"/>` using the existing ToolBar SeparatorStyleKey.

**`btn_circleRoi` spec (D-15):**
- Type: `ToggleButton`, `Style="{StaticResource RoiToggleButtonStyle}"`
- Content: `"Circle ROI"` (English, 13-char, fits MinWidth=80 without truncation)
- Dimensions: `MinWidth="80" Height="28" Padding="12,4"`
- Typography: `FontSize="13" FontWeight="SemiBold"`
- Margin: `Margin="0,0,4,0"` (matches Rect/Polygon spacing)
- Placement: Immediately after `btn_polygonRoi` in the StackPanel
- `IsEnabled` binding: `True` when selected node is FAI AND the FAI contains a `CircleDiameterMeasurement` (or the selected node itself is a `CircleDiameterMeasurement`). Otherwise `False`. Mirror the existing FAI-only activation used by Rect/Polygon (D-18).

**`btn_teachDatum` spec (D-01):**
- Type: `ToggleButton`, `Style="{StaticResource RoiToggleButtonStyle}"`
- Content: `"Teach Datum"` (English)
- Dimensions: `MinWidth="100" Height="28" Padding="12,4"` — width raised to 100 (vs 80) because "Teach Datum" is 11 chars and gets visually cramped at 80. 100 still fits the 4-unit scale.
- Typography: `FontSize="13" FontWeight="SemiBold"`
- Margin: `Margin="0,0,4,0"`
- Placement: Right side of the first Separator, i.e. after the Rect/Polygon/Circle group
- `IsEnabled` binding: `True` if and only if selected node is `ENodeType.Datum`. Otherwise `False` (per D-01).

**`btn_testFai` spec (D-23) — placement decision (CONTEXT.md left as Claude's Discretion at D-24):**
- Decision: **place in the canvas toolbar, in group 3, immediately right of `btn_calibrate`.** Rationale: D-24 calls Test FAI a "tuning tool in the same line as Teach Datum"; keeping it on the canvas toolbar co-locates it with the image-in-view workspace (which is where the user is looking when they press it). Placing it inside InspectionListView would force the user's eye to jump between the tree panel and the canvas. Both Teach Datum and Test FAI are operations performed on the currently displayed image.
- Type: `Button` (NOT ToggleButton — one-shot action, identical semantic to Calibrate)
- Content: `"Test FAI"`
- Dimensions: `MinWidth="80" Height="28" Padding="12,4"`
- Typography: `FontSize="13" FontWeight="SemiBold"`
- Colors: `Background="Transparent" Foreground="#FFAAAAAA" BorderBrush="#FF555555"` (idle, matches RoiToggleButtonStyle idle — intentionally NOT accent blue, because Calibrate already owns accent-as-idle and a second always-accent button would dilute the visual hierarchy). On IsMouseOver: `Background="#3A3A4A"` Foreground="White". Use a local ControlTemplate that mirrors the RoiToggleButtonStyle idle/hover triggers but without the IsChecked trigger.
- Margin: `Margin="4,0,0,0"`
- `IsEnabled` binding: `True` if selected node is FAI or Measurement AND the parent Shot has an image in memory. Otherwise `False` (per D-22/D-23).

### InspectionListView TreeListBox — node status badge (D-20)

Insert an 8×8 `Ellipse` at the start of the node's `DataTemplate`, before the existing text label:

```xml
<!-- TreeListBox HierarchicalDataTemplate (extended) -->

    <Ellipse Width="8" Height="8" Margin="0,0,6,0"
             VerticalAlignment="Center"
             Fill="{Binding StatusColor}"/>

Prefer the `StatusColor` brush property for testability.

## Interaction Contract

### WR-RT-01: Circle ROI drawing (D-14–D-19)

| Step | Trigger | Response |
|------|---------|----------|
| 1 | User selects FAI node containing a CircleDiameterMeasurement (or the Measurement node itself) | `btn_circleRoi.IsEnabled = true` |
| 2 | User clicks `btn_circleRoi` | `btn_circleRoi.IsChecked = true`; `MainView._canvasMode = ECanvasMode.CircleRoi`; canvas cursor → crosshair; `label_drawHint.Visibility = Visible`, text = "중심을 클릭 후 드래그하여 반지름을 지정하세요", Foreground = `#FFAAAAAA` (info) |
| 3 | User clicks on canvas (MouseDown) | `HalconViewerControl.StartCircleDrawing(centerRow, centerCol)`; `_circleDraftCenter` stored; draft center cross (red) rendered |
| 4 | User drags (MouseMove) | Radius = Euclidean distance from draft center to cursor; preview circle (red outline, red center cross) rerendered each move |
| 5 | User releases (MouseUp) | `HalconViewerControl.CommitActiveCircle()` → raises `CircleDrawingCompleted(centerRow, centerCol, radius)` event |
| 6 | `CircleDrawingCompleted` handler | `MainView.CommitCircleRoi(...)` writes to selected Measurement's `Circle_Row/Col/Radius` (D-17); also updates the FAI's `RoiDefinition.Shape = RoiShape.Circle`, `CenterRow/CenterCol/Radius` (D-16) as the canvas edit buffer; commits Halcon display → circle outline turns `lime green` (committed, unselected) or `yellow` (if this is the currently selected node); `btn_circleRoi.IsChecked = false`; `label_drawHint.Visibility = Collapsed`; node status badge transitions Red→Green |
| — | User presses Escape during draw | Draft cleared; `btn_circleRoi.IsChecked = false`; `label_drawHint.Visibility = Collapsed`; no write to Measurement fields |

### WR-RT-03: Datum teaching (D-01–D-13)

| Step | Trigger | Response |

| 6b | TryTeachDatum returns false | `label_drawHint` error = `"Line1 검출 실패: " + error` or equivalent (`#FFF87171`); the two ROI rectangles remain drawn so user can adjust; `btn_teachDatum.IsChecked = false`; user can click `btn_teachDatum` again to retry (step 3a flow — since IsConfigured still false, no confirmation modal, straight re-enter) (D-12) |

**Destructive actions in Phase 11:**
1. **Datum 재티칭** (overwrites Line1/Line2/RefOrigin/RefAngle) — confirmation required via `CustomMessageBox.ShowConfirmation` (D-04). Yes/No copy above.
2. **Existing Circle ROI redraw** — silently overwrites previous Circle_Row/Col/Radius. NOT flagged as destructive per project convention (Phase 02 treated ROI redraw the same way — non-modal, recoverable via recipe reload). No confirmation.

**Empty state copy:**
- Datum node with no image loaded: canvas shows existing `NO Image` label (unchanged); status bar hint drives the user forward.
- FAI with CircleDiameterMeasurement and `Circle_Radius<=0`: node badge red; status bar hint says to draw Circle ROI.
- No test executed yet: `label_drawHint.Visibility = Collapsed` — intentionally empty (not a state the user needs a message for).

**Error state copy pattern:** `{problem}: {recovery action}` — all error hints include both the detected problem and what to do next. Examples: "Shot '{name}'을 먼저 Grab하세요" (problem=missing image, recovery=Grab); "Datum 티칭 실패: insufficient edge points" (problem=detection failed, recovery implied by `out error` detail + retained ROIs for repositioning).
