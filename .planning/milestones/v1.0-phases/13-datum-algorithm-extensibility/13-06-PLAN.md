**라우팅 가드 핵심:**
- ParamEditor 의 routed event 는 SelectedObject 가 어떤 ParamBase 든(FAIConfig/MeasurementBase/DatumConfig) bubble 로 fire 됨 — DatumConfig 인지 + IsConfigured + LastTeachSucceeded 체크 후 분기. **DatumConfig 가 아닌 경우 즉시 return** (FAI/측정 PropertyGrid 편집에는 영향 0).

**LostFocus 트리거 의미:**
- WPF TextBox 의 default `UpdateSourceTrigger` = `LostFocus`. 즉 사용자가 다른 셀로 포커스 이동 시점에 binding 이 source(DatumConfig 필드) 로 push.
- 따라서 `TextBoxBase.LostFocusEvent` routed event 가 fire 될 시점에는 이미 DatumConfig 필드가 새 값으로 갱신된 상태 — 그 직후 `InvokeTryTeachDatumForEdit` 호출이 적절.
- ComboBox(EdgeDirection/EdgePolarity dropdown) 는 `Selector.SelectionChangedEvent` 에서 binding push 가 즉시 일어남 — 동일 시점.

**Spurious trigger 허용:**
- LostFocus 는 값 변경 없이도 fire 됨(탭 이동 등). 이 경우에도 InvokeTryTeachDatumForEdit 가 한 번 더 호출되지만 동일 파라미터로 결과도 동일 — 시각적 변화 없음. CPU 비용은 < 50ms (Halcon edge measurement). UX 허용.
- 더 정밀한 "값 변경 detection" 은 binding 단위 hook 필요 → 변경 면적 과대 → Phase 14 일임.
