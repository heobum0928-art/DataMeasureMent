## Layout Contract

Source: CONTEXT.md D-03, D-04, RESEARCH.md Pattern 2.

```
MainWindow
├── TitleBar (Row 0, Height=26)         ← unchanged
├── MenuBar  (Row 1, Height=80)         ← unchanged
├── Content area (Row 2, 78*)
│   └── MainView.xaml [REPLACE CONTENTS]
│       ├── Left column (25*): TreeView panel
│       │   ├── Toolbar (Height=36): Add(+) | Remove(−) | Edit (pencil)
│       │   └── TreeView (Height=*): Shot nodes → FAI leaf nodes
│       ├── Vertical GridSplitter (Width=5)
│       └── Right column (75*): Grid
│           ├── Top row (6*): MainResultViewerControl canvas
│           ├── Horizontal GridSplitter (Height=5)
│           └── Bottom row (4*): DataGrid result table
└── StatusBar (Row 3, Height=26)        ← unchanged
```

**Rationale:** The 25/75 column split and 6/4 canvas/table row split match the visual weight priorities: canvas is the primary workspace, tree is navigation, table is secondary reference. These ratios follow D-03 and the existing MainWindow pattern (75*/25* columns already used).

**Constraint:** `MainWindow.xaml` itself is NOT modified. Only `MainView.xaml` contents are replaced.

## Spacing Scale

WPF uses device-independent pixels (DIPs). All values are multiples of 4.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-text gap in tree nodes, toolbar button inner padding |
| sm | 8px | DataGrid row padding, toolbar separator margin |
| md | 12px | Canvas border Padding (matches existing MainResultViewerControl Padding="4"; toolbar Margin) |
| lg | 16px | Tree panel outer margin, DataGrid column header padding |
| xl | 24px | Section separation between toolbar and tree body |
| 2xl | 32px | Not used in Phase 1 |
| 3xl | 64px | Not used in Phase 1 |

**Exceptions:**
- Toolbar button icons: Width=32 Height=32 (matches existing `/Resource/` icon buttons in InspectionListView — maintains visual consistency)
- Toolbar row Height=36 (matches existing InspectionListView toolbar row: `<RowDefinition Height="36">`)
- GridSplitter width/height=5 (matches existing MainWindow.xaml GridSplitter Width="5" / Height="5")

Source: Extracted from `InspectionListView.xaml`, `MainWindow.xaml`, `MainResultViewerControl.xaml`.

## Color

Source: Extracted from `MainResultViewerControl.xaml`, `MainView.xaml`, `MainWindow.xaml`, `MessageBoxWindow.xaml`, RESEARCH.md DataTrigger pattern. Confirmed by D-06.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#FF303030` | Canvas area background (matches existing MainView Grid.Row="1" Background); TreeView panel background |
| Secondary (30%) | `#FF1E293B` (matches `#0F172A` darkened slightly) — use `#0F172A` for canvas inner, `#1C2333` for tree/table panel | TreeView panel, DataGrid background, MainView outer Grid |
| Accent (10%) | `#FFc7e6fd` (light blue — existing TitleBar color) | Selected tree node highlight background; Toolbar button hover state |
| Destructive | `#FFcc0000` (solid red, stronger than `#FF402020`) | Delete confirmation dialog border (matches MessageBoxWindow red BorderBrush); NG result row background |

**Correction note:** The existing app uses a mixed light/dark theme at the window level (Beige/Lavender outer shell) but a dark theme inside the content canvas. Phase 1 introduces a new panel (TreeView + DataGrid) that sits inside the same content area. Use dark panel backgrounds (`#FF2D3748` for TreeView panel, `#FF1A202C` for DataGrid rows) to stay consistent with the canvas dark theme. Do NOT use Beige or Lavender for new Phase 1 panels.

**OK/NG row colors (D-06):**

| State | Background | Text foreground |
|-------|-----------|----------------|
| OK (IsPass=true) | `#FF204020` | `#FFFFFFFF` |
| NG (IsPass=false) | `#FF402020` | `#FFFFFFFF` |
| No result (HasResult=false) | `#FF2D2D2D` | `#FF8899AA` |

Source: RESEARCH.md DataTrigger pattern (Pattern 3), D-06.

**Accent reserved for:**
- Selected TreeView node highlight background
- Toolbar button background on mouse-hover (replaces `LightSkyBlue` currently used — use `#FFc7e6fd` instead for consistency with TitleBar)
- Active/selected DataGrid row selection highlight

Never apply accent to: unselected tree nodes, DataGrid cells, canvas borders, status text.

## Interaction Contract

### TreeView Interaction

| Trigger | Response |
|---------|----------|
| User expands Shot node | FAI child nodes appear (HierarchicalDataTemplate auto-renders `FAIItems`) |
| User clicks Shot node | Canvas loads Shot image via `MainResultViewerControl.LoadImage(HImage)`; DataGrid refreshes to show that Shot's FAI result rows |
| User clicks FAI leaf node | DataGrid scrolls to and selects the corresponding FAI row; ROI highlight deferred to Phase 2 |
| No node selected (initial state) | Canvas shows "No Image" message; DataGrid shows empty state message |

**TreeView SelectedItem binding:** Use `SelectedItemChanged` event in code-behind to set `ViewModel.SelectedNode` (two-way binding on `SelectedItem` is not supported in WPF TreeView — see RESEARCH.md Pitfall 1).

### Toolbar Interaction (D-08)

| Button | Selected node is null | Selected node is Shot | Selected node is FAI |
|--------|-----------------------|-----------------------|---------------------|
| Add (+) | Opens "Shot 이름 입력" dialog → adds new Shot at end | Opens "FAI 이름 입력" dialog → adds FAI under selected Shot | Opens "FAI 이름 입력" dialog → adds FAI under parent Shot |
| Remove (−) | Disabled (gray, Opacity=0.5) | Shows delete confirmation for Shot | Shows delete confirmation for FAI |
| Edit (pencil) | Disabled (gray, Opacity=0.5) | Opens "Shot 이름 수정" dialog with current name pre-filled | Opens "FAI 이름 수정" dialog with current name pre-filled |

**Disabled state:** Buttons use `IsEnabled="False"` + `Opacity="0.5"` (matches existing `disabledStyle` in InspectionListView.xaml).

### DataGrid Interaction

| Trigger | Response |
|---------|----------|
| User clicks FAI row | Row highlights with selection color; D-07 ROI highlight deferred to Phase 2 (stub hook in code-behind) |
| Inspection not yet run | Row shows "—" in distance/judge columns (requires `HasResult` field in FAIResultRow) |
| Inspection complete | Row background is `#FF204020` (OK) or `#FF402020` (NG) per D-06 |

**DataGrid settings:** `IsReadOnly="True"`, `SelectionUnit="FullRow"`, `AutoGenerateColumns="False"` (per RESEARCH.md Pitfall 3).

## States Inventory

Every element must handle all applicable states. Executor must implement all rows.

### TreeView

| State | Rendering |
|-------|-----------|
| Empty (no shots) | TreeView shows no nodes; status label below tree: "레시피에 Shot이 없습니다." |
| Shot node collapsed | Shot row with expand arrow; no FAI children visible |
| Shot node expanded | Shot row + indented FAI child rows |
| Shot node selected | Row background: accent `#FFc7e6fd`; text foreground: `#FF000000` |
| FAI leaf node selected | Row background: accent `#FFc7e6fd`; text foreground: `#FF000000` |
| No selection | All nodes default background `transparent` |

### Canvas

| State | Rendering |
|-------|-----------|
| No image | Black background (`#0F172A`); `label_message` shows "NO Image" in Red at FontSize 20 |
| Image loaded | Halcon HWindowControlWPF displays image; `label_message` cleared |
| Image load error | `label_message` shows "이미지 로드 실패" in Red |

### DataGrid

| State | Rendering |
|-------|-----------|
| No Shot selected | Empty; show no rows |
| Shot selected, no FAI | One placeholder row: `FAIName="FAI 없음"`, distance="—", spec="—", judge="—"; background `#FF2D2D2D` |
| Shot selected, FAI exists, no result | Row per FAI; distance="—", judge="—"; background `#FF2D2D2D` |
| Shot selected, FAI measured OK | Background `#FF204020`, JudgeText="OK" |
| Shot selected, FAI measured NG | Background `#FF402020`, JudgeText="NG" |
| Row selected | WPF default selection highlight on top of row background |

## Constraints & Anti-Patterns

Directly from RESEARCH.md. Executor must NOT violate these.

| Constraint | Enforcement |
|------------|------------|
| Do NOT keep TabControl | MainView.xaml TabControl is fully replaced with 3-zone Grid layout |
| Do NOT reuse existing `NodeViewModel` | It is `ESequence`/`EAction` based; create `ShotNodeViewModel` and `FAINodeViewModel` |
| Do NOT bind `DataGrid.ItemsSource` directly to `FAIConfig` list | Use `FAIResultRow` DTO (wraps FAIConfig; avoids PropertyTools annotations leaking into DataGrid) |
| Do NOT use `List<T>` for `ShotNodeViewModel` collection | Use `ObservableCollection<ShotNodeViewModel>` so TreeView reacts to CRUD changes |
| Do NOT bind `TreeView.SelectedItem` two-way | Use `SelectedItemChanged` event to push to ViewModel (WPF read-only DependencyProperty) |
| Do NOT use `DataGrid IsReadOnly=False` | Always `IsReadOnly="True"` + `SelectionUnit="FullRow"` in result table |
| Do NOT call `HImage.Dispose()` before `MainResultViewerControl.LoadImage()` returns | Follow `MainView.GrabAndDisplay` pattern; viewer retains reference internally |
| Do NOT make Shot delete atomic at model level only | Synchronize `_shots ObservableCollection` and `InspectionRecipeManager.RemoveShot(index)` in one ViewModel method call |
| Do NOT mix Allman and K&R brace style within one file | New files follow Allman (consistent with new Halcon code); edited files follow file's existing style |
| Do NOT use C# 8.0+ features | C# 7.2 only (no `record`, no nullable reference types, no switch expressions) |
