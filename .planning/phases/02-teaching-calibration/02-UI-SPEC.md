## Interaction Contract

### TCH-01: ROI Overlay on FAI Selection

Trigger: User selects FAI node in TreeListBox OR row in DataGrid.
Response:
1. InspectionViewModel converts selected FAIConfig → RoiDefinition via `ToRoiDefinition()`.
2. Calls `MainResultViewerControl.UpdateDisplayState(allRois, selectedRoiId, null, null)`.
3. HalconDisplayService renders all ROIs in green; selected ROI in yellow.
4. Edge direction arrow rendered at ROI center (white, 10-15px).

States:
- No FAI selected: no ROI overlay, canvas shows image only.
- FAI selected but ROI not taught (Length1=0 or Length2=0): no ROI overlay; canvas shows "ROI not set" text at bottom-left in `#FFAAAAAA`.

### TCH-01: Rect ROI Drawing

Trigger: User clicks "Rect ROI" button (enters rect-draw mode).
Response:
1. Button turns active (blue background).
2. Canvas cursor changes to crosshair.
3. Mouse drag on canvas → `HalconViewerControl.StartRectangleDrawing()` → rubber-band rect in red (`#FF0000`).
4. Mouse release → `CommitActiveRectangle()` → ROI committed, outline turns yellow (selected).
5. FAIConfig.ROI_Row/Col/Phi/Length1/Length2 updated.
6. "Rect ROI" button returns to idle state automatically.
7. InspectionListView PropertyGrid refreshes to show updated values.

Cancel: Press Escape during draw → discard draft, button returns to idle.

### TCH-01: Polygon ROI Drawing

Trigger: User clicks "Polygon ROI" button (enters polygon-draw mode).
Response:
1. Button turns active (blue background).
2. Canvas cursor changes to crosshair.
3. Left-click places a point (red dot). Point count badge increments.
4. Rubber-band line follows cursor from last point.
5. At 20 points: no more left-clicks accepted; badge shows "20 / 20 pts MAX".
6. Right-click (minimum 3 points placed): polygon completed. Outline turns blue.
7. Polygon serialized to FAIConfig string field as "x1,y1;x2,y2;..." format (D-15).
8. "Polygon ROI" button returns to idle.

Cancel: Escape key during draw → discard all placed points, button returns to idle.

### TCH-02: ROI Save

Trigger: Existing recipe Save (context menu "Save" in InspectionListView, or Ctrl+S equivalent).
Response: FAIConfig fields (ROI_Row/Col/Phi/Length1/Length2 + PolygonPoints string) serialized via ParamBase.Save → INI file. No additional UI action needed beyond existing save path (D-14).

Confirmation: Status bar text in InspectionListView updates to "Saved: [recipe name]" for 2 seconds, then reverts. No modal dialog.

### ALG-03: Calibration Flow

Trigger: User clicks "Calibrate" button.
Response — 2-point measurement mode:
1. Button label changes to "Pick Point 1" (disabled until canvas has image).
2. Canvas cursor changes to crosshair with circle cursor icon.
3. User clicks point 1 → small white circle marker rendered at click position. Button label: "Pick Point 2".
4. User clicks point 2 → second marker; rubber-band line between points with pixel distance shown as "NNN px" label (white, 13px, next to midpoint).
5. `TextInputBoxWindow` opens with title "실제 거리 입력" and prompt "두 점 사이의 실제 거리(mm)를 입력하세요:".
6. User enters mm value and clicks OK.
7. mm/pixel = input_mm / pixel_distance calculated and stored to CameraSlaveParam.PixelResolution.
8. Canvas shows confirmation text "1 px = X.XXX mm 적용됨" (white, 13px, bottom-left of canvas) for 3 seconds.
9. Calibrate button returns to normal state; markers cleared.

Cancel: Escape at any point → markers cleared, button returns to normal.
Validation: Input must be a positive double. If invalid, TextInputBoxWindow shows inline error "유효한 숫자를 입력하세요" below TextBox (red, 12px) and stays open.
