**P-1 (D-03): `??` 단일** — `var v = a ?? b;` →
```csharp
// Before:
Id = FAIName ?? "FAI",

// After (object initializer 안이라 inline 임시변수 못 씀 → 메서드/생성자 진입부에서 분해):
string idValue = "FAI";
if (FAIName != null) idValue = FAIName;
// ...
Id = idValue,
```
초기화 블록 안의 `??` 가 4 곳 이상 모이는 경우, 객체 생성 위에 임시 변수 그룹을 만들고 초기화 블록은 깨끗하게 유지.

**P-2 (D-03): `obj?.Field ?? defaultValue`** →
```csharp
// Before:
var v = obj?.Field ?? defaultValue;

// After:
var v = defaultValue;
if (obj != null) v = obj.Field;
```

**P-3 (D-02): `event?.Invoke(this, e)`** → 멀티스레드 경주 안전:
```csharp
// Before:
RoiGeometryChanged?.Invoke(this, args);

// After:
var handler = RoiGeometryChanged;
if (handler != null) handler(this, args);
```

**P-4 (D-01): `?:` 삼항** →
```csharp
// Before:
var s = bIsOk ? "OK" : "NG";

// After:
string s = "NG";
if (bIsOk) s = "OK";

// 또는 early return / 중첩 if 가 더 명확하면 그것을 우선.
```

**P-5 (D-04 예외 — LINQ tail `?.` 유지):**
```csharp
// 그대로 둔다 (변환 X):
var v = list.FirstOrDefault(x => x.Name == n)?.Value;
```

**P-6 (D-05 예외 — expression-bodied 유지):**
```csharp
// 그대로 둔다 (변환 X):
public string Name => _name;
```
