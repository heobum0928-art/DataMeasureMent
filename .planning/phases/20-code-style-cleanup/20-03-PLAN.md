**P-3 (D-02): event handler — 본 파일의 핵심 패턴 (`?.Invoke` 12 곳):**
```csharp
// Before:
RoiGeometryChanged?.Invoke(this, args);

// After:
var handler = RoiGeometryChanged;
if (handler != null) handler(this, args);
```
12 이벤트 모두 동일 패턴. handler 임시변수 명명 규칙 (W4 — 같은 스코프 다중 발신 시 충돌 방지):
- 첫 번째 발신: `handler` 단순 명명 가능.
- 같은 스코프(동일 메서드 또는 동일 if 블록) 의 두 번째 이상 발신: 이벤트명 camelCase 사용 (예: `pointerHandler`, `redrawHandler`, `mouseDownHandler`). 또는 매 이벤트마다 `{ ... }` 새 스코프 블록으로 분해 시 `handler` 변수 재선언 가능.
- 12 위치 중 line 1369 / 1386 / 1694 (PointerInfoChanged ×4 등 같은 메서드 내 다중 발신) 가 본 규칙의 핵심 적용 대상.

**P-3a (이벤트 핸들러 분해 시 추가 라인 — 코드 가독성):**
- 이벤트 발신이 메서드 끝의 한 줄이면 P-3 분해.
- 이벤트 발신이 luxiously 짧은 메서드 (단 한 줄짜리) 라면 그래도 P-3 분해 — D-01 의 일관성 우선.

**P-2 강화 (UI null safety):**
```csharp
// Before:
var name = SelectedItem?.Name ?? "Untitled";

// After:
string nameValue = "Untitled";
if (SelectedItem != null) nameValue = SelectedItem.Name;
var name = nameValue;

// 또는 더 간결:
string nameValue = "Untitled";
if (SelectedItem != null) nameValue = SelectedItem.Name;
```
