**C# 7.2 한정 (W1 — .NET Framework 4.8):** `??=` (null-coalescing assignment), `is { }` property pattern, switch expression (`x switch { ... }`), tuple deconstruction with discards 사용 금지. 명시적 if/else / 임시변수 + null 체크 패턴만 허용 (D-01 강제).

**P-11 (UI dispatcher null safety):**
```csharp
// Before:
Dispatcher?.Invoke(() => DoStuff());

// After:
var disp = Dispatcher;
if (disp != null) disp.Invoke(() => DoStuff());
```
WPF Dispatcher 가 null 인 경우는 거의 없지만 (UserControl 의 Dispatcher 는 항상 존재), 패턴 일관성 위해 동일 분해.

**P-12 (Datum 티칭 상태머신 분기):**
EDatumTeachStep switch 안의 `?:` 가 DatumConfig 필드 접근에 쓰이는 경우, 분해 시 step 라벨 명확:
```csharp
// Before:
var rect = (step == EDatumTeachStep.HorizontalA) ? datum.HorizontalA_Rect : datum.HorizontalB_Rect;

// After:
HRect rect = datum.HorizontalB_Rect;
if (step == EDatumTeachStep.HorizontalA) rect = datum.HorizontalA_Rect;
```
