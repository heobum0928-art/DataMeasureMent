<objective>
`EdgeToLineDistanceMeasurement` 의 Point ROI 를 measurement 단위로 캔버스 드로잉 티칭할 수 있게 배선하고, 1개 FAI 가 N개 measurement(다점 P1/P2/…)를 갖는 유동 구조를 UI 로 운용 가능하게 한다.

**문제:** Phase 23 의 `EdgeToLineDistanceMeasurement` 는 측정에 `Point_*` ROI 필드를 쓰지만, 캔버스 드로잉 → `Point_*` 배선이 없다. 현재 `CommitRectRoi` 는 `FAIConfig.ROI_*` 로만 쓴다. measurement 별 ROI 티칭 경로가 끊겨 있어 A1~A5 처럼 1 FAI = N measurement(다점) 구조에서 각 점의 ROI 를 개별 티칭할 수 없다.

**해결 (D-01~D-05):**
- **D-01:** 기존 Rect ROI 버튼 컨텍스트를 재사용한다 — 신규 toolbar 버튼 없음, `ECanvasMode` 신규 값 없음. `CommitRectRoi` 가 새 `_editingMeasurement` 상태로 분기: Measurement 노드 선택 시 `EdgeToLineDistanceMeasurement.Point_*` 로 write-back, FAI 노드 선택 시 기존대로 `FAIConfig.ROI_*`. 작동하는 analog = `CommitCircleRoi` + `_editingCircleMeasurement` (이미 동작 중인 measurement별 ROI 티칭 경로).
- **D-02:** 배선은 `EdgeToLineDistanceMeasurement` 만 대상. `PointToLineDistanceMeasurement` 등 다른 Point ROI 타입은 무수정.
- **D-03:** FAI 노드 선택 시 그 FAI 의 모든 measurement Point ROI 동시 렌더 — `GetCurrentFAIRois` 가 Measurement 단위 ROI 도 수집.
- **D-04/D-05:** 다점 치수 = "1 FAI = N measurement" 유동 구조. `FAIConfig.Measurements` (`List<MeasurementBase>`) 가 이미 N개를 지원. Add/Remove Measurement UI 도 Phase 6 에서 이미 동작 — 이 plan 의 유일한 UI 변경은 Measurement 노드 선택 시 `btn_rectRoi` 활성화.

Purpose: ALG-01 carry-over 완결의 핵심. ROI 티칭이 배선돼야 사용자가 A1~A5 다점 measurement 의 ROI 를 셋업할 수 있고 Plan 03 의 실데이터 측정 검증이 가능해진다.
Output: `MainView.xaml.cs` + `InspectionListView.xaml.cs` 수정 (신규 파일 없음).
