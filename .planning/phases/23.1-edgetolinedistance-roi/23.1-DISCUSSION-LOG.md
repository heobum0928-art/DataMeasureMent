## ROI 티칭 배선

### Q1. EdgeToLineDistance measurement 의 Point ROI 를 사용자가 어떻게 그리게 할까요?

| Option | Description | Selected |
|--------|-------------|----------|
| 기존 Rect 버튼 재사용 | Measurement 노드 선택 시 기존 Rect ROI 버튼이 measurement.Point_* 로, FAI 노드 선택 시 FAIConfig.ROI_* 로 컨텍스트 분기. 버튼 추가 0. | ✓ |
| 별도 Point ROI 버튼 신규 | Measurement 선택 시만 활성화되는 전용 toolbar 버튼 추가. | |

**User's choice:** 기존 Rect 버튼 재사용
**Notes:** CircleDiameterMeasurement 의 CommitCircleRoi→Circle_* 가 작동하는 analog.

### Q2. ROI→Point_* 배선을 어느 measurement 타입까지 적용할까요?

| Option | Description | Selected |
|--------|-------------|----------|
| EdgeToLineDistance 만 | 23.1 범위 한정. Point_* 가진 다른 타입은 무수정 — 회귀 위험 최소. | ✓ |
| Point_* 보유 전체 타입 | PointToLineDistance/PointToPointDistance 등도 동일 배선. 범위 확대. | |

**User's choice:** EdgeToLineDistance 만

### Q3. 다점 FAI 티칭 중 캔버스에 ROI 를 어떻게 표시할까요?

| Option | Description | Selected |
|--------|-------------|----------|
| FAI 의 모든 measurement ROI 동시 | FAI 노드 선택 시 모든 measurement Point ROI(P1·P2) 동시 표시, Measurement 선택 시 강조. | ✓ |
| 선택된 measurement ROI 만 표시 | 한 번에 1개 ROI. 단순하나 다점 티칭 시 다른 점 안 보임. | |

**User's choice:** FAI 의 모든 measurement ROI 동시 표시

## 다점 치수 구조

### Q1. A1~A5 중 어느 FAI 가 다점(점 2개) 측정인가요?

| Option | Description | Selected |
|--------|-------------|----------|
| A3·A4 만 2점 | SOP 요약표 '2x' 표기 기준. A1·A2·A5 단점. | |
| A1/A3/A4/A5 2점 | 상세 페이지(p.39~43) 분석 기준. A2 만 단점. | |
| 캐퍼빌리티만 중요 | 정확한 집합은 레시피 셋업 시 결정. | ✓ (Other) |

**User's choice:** (자유 입력) "이거는 유동적으로 하면 안 되나 3개든 2개든 1개든 바뀔 수도 있어"
**Notes:** FAI당 measurement 개수를 유동(1/2/3+)으로 — 레시피 데이터로 결정, 코드는 List<MeasurementBase> N개 그대로, 특정 집합 하드코딩 안 함. 기존 아키텍처와 일치하며 SOP 표기 불일치도 해소.

## EdgeSelection 마이그레이션

### Q1. 기존 레시피에 EdgeSelection=First 로 저장된 EdgeToLineDistance measurement 를 어떻게 처리할까요?

| Option | Description | Selected |
|--------|-------------|----------|
| All 고정 + PropertyGrid 숨김 | EdgeSelection 내부 'All' 고정, ICustomTypeDescriptor 로 숨김. 레거시 'First' 무시 → 마이그레이션 불필요, 결함 재발 차단. | ✓ |
| 노출 유지 + 로드 시 자동 교정 | EdgeSelection 노출 유지(기본 All), 로드 시 First→All 자동 교정 + 로그. 재발 가능. | |
| 마이그레이션 없음 | 신규만 All 기본값, 레거시 INI 수동 수정. 구버전 레시피 여전히 실패. | |

**User's choice:** All 고정 + PropertyGrid 숨김
**Notes:** EdgeToLineDistance 는 line fit 에 edge 점 ≥2 필요 → 'First'(1점)는 항상 실패하므로 유효한 선택이 아님. Phase 19 DynamicPropertyHelper 재사용.

### Q2. 이번 phase 의 EdgeSelection 관련 변경을 어느 범위까지 적용할까요?

| Option | Description | Selected |
|--------|-------------|----------|
| EdgeToLineDistance 만 | 23.1 범위 한정. 다른 Point 기반 타입의 EdgeSelection 무수정. | ✓ |
| 전체 measurement 타입 점검 | 모든 타입 EdgeSelection 동작 점검/통일. 회귀 검증 폭 증가. | |

**User's choice:** EdgeToLineDistance 만
