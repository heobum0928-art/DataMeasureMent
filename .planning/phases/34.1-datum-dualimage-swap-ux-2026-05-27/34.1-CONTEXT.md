### Area A — 회귀 게이트 분기 정책 (D-34.1-07 보강 위 참조)

### Area B — 수동 swap UI 구현 메커니즘

- **D-34.1-08**: Datum 노드 이동 시 swap 상태 초기화 = 기본 표시 = **가로축**(TeachingImagePath). 세션 한정, INI 미저장. (Why: 사용자 워크플로우의 기본 시작점이 가로축이며, 상태 보존은 다른 Datum 간 일관성 위험 → 단순화로 회피.)
- **D-34.1-09**: swap 버튼 Visibility = `AlgorithmType == VerticalTwoHorizontalDualImage` 일 때만 Visible. 1-image (TLI/CTH/VTH) 는 Collapsed. AlgorithmType 변경 시 즉시 Visibility 업데이트. (Why: 1-image 는 swap 할 이미지 자체가 없음 → UI 노출 시 혼란.)

### Area C — 수동 swap 시 ROI / wizard 상태 보존

- **D-34.1-10**: ROI 표시 정책 = 현재 표시 이미지에 속하는 ROI 만 캔버스에 그림. 가로 이미지 표시 시 = `Datum.HorizontalA` + `Datum.HorizontalB` ROI 만, 세로 이미지 표시 시 = `Datum.Vertical` ROI 만. ROI 데이터 자체는 DatumConfig 에 그대로 보존 — 가시성(Visibility/Opacity)만 토글. (Why: 가로 이미지에 세로축 ROI 보이면 사용자가 "여기서 그릴 수 있나?" 혼동 + ROI 시맨틱 위반.)
- **D-34.1-11**: Wizard step 라벨 ↔ 배지 의미 분리 = swap 시 wizard step 라벨 변경 **안 함**. step 라벨 = "편집 의도(다음 그릴 ROI)", 배지 = "현재 화면(어느 이미지)". (Why: 사용자가 Vertical step 진행 중에 가로 이미지로 잠깐 토글하여 결합 결과/HA/HB 검증 가능해야 함 — 라벨과 배지를 강제 동기화하면 이 패턴 막힘.)
- **D-34.1-12**: Idle (wizard 외) Datum 노드 선택 상태에서도 동일 동작 = 두 이미지 자유 토글, ROI 는 해당 축 만 표시.

### Area D — 배지 디자인 + UAT 6/7 합격 기준 + SIMUL 의사 페어 이미지

- **D-34.1-13**: SIMUL 의사 듀얼 이미지 페어 = 기존 `Cal_Image/` 하위 SIMUL 이미지 재활용. 가로축 = Top 카메라 이미지 1장 + 세로축 = `Cal_Image/Side/` 의 Side 카메라 이미지 1장 조합. 신규 폴더 = `Cal_Image/DualImageTest/` (executor 가 최종 파일명/경로 확정). 두 이미지 해상도/orientation 불일치 검증은 본 phase 범위 외 (Phase 34 deferred 와 동일 — operational discovery 시 별도 phase).
- **D-34.1-14**: 배지 디자인 구체화 — 가로축 색상 = `#1976D2` (Material Blue 700), 세로축 색상 = `#F57C00` (Material Orange 800), 폰트 크기 14px, 텍스트 = "가로축" / "세로축" (이모지 없음, Border + TextBlock 단순 컨트롤), 캔버스 우상단 마진 12px. Visibility = `AlgorithmType == VerticalTwoHorizontalDualImage` 일 때만 Visible (D-34.1-09 와 동기화).
- **D-34.1-15**: UAT Test 6 (동기화) PASS 기준 = 수동 swap 토글 시 (a) 배지 텍스트, (b) 배지 색상, (c) ROI 가시성 **3자 동시 전환** + 시각적 지연 없음. 자동 swap (wizard step 진행) 시에도 동일 3자 동시 전환.
