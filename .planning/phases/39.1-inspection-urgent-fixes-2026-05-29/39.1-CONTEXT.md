### Item #1 — CircleDiameter Measurement 파라미터 누락 (긴급, 알고리즘)

**사용자 원문:**
> CircleDiameter MeasureMent 알고리즘 수정 필요. 원에서 360도 돌릴 때 몇 도씩 돌려야 하는 파라미터 누락, Length1, Length2 파라미터 누락으로 각각 에지를 뽑아야 하는 파라미터들 누락

**분류:** 코드 + UI (PropertyGrid 노출)

**관련 코드/기록:**
- Phase 28-01 (Plan 38d86186a): `EdgeOptionLists.FaiCirclePolarStepDeg = 10.0`, `RectL1Ratio = 0.02`, `RectL2Ratio = 0.02` 상수 추가됨 (Datum CTH 동등성)
- Phase 28-02: `CircleDiameterMeasurement.Circle_RadialDirection` (string, default "") + `Circle_RadialDirectionList` 추가
- 현재 상태 추정: 사용자가 PropertyGrid 에서 polar step / Length1 / Length2 등 파라미터를 직접 조정 못 함 (필드 미노출 또는 기본 상수만 사용)

**예상 작업:**
- `CircleDiameterMeasurement` 에 polar step (deg) / strip Length1 / strip Length2 PropertyGrid 필드 추가
- `EdgeOptionLists.FaiCircle*` 상수를 default 로 유지, 사용자 override 가능하도록 인스턴스 필드화
- INI 직렬화 호환 (ParamBase reflection)

**Anti-goals:**
- Datum CTH (CircleTwoHorizontal) 의 polar 동작 변경 0 (Phase 28 D-03 동등성 가드)
- 기존 INI 회귀 0 (default 값 = 현재 상수 값)

**분류:** 코드 알고리즘 (수정)

**관련 코드/기록:**

- Phase 31-D-04: `ComputeProjectionDistance` static 추출, 신규 타입 6종 재사용
- Phase 31 Task 3(C) 보류: `EdgeToLineDistanceMeasurement` 내부 projection_pl 블록 유지 (overlay footRow/footCol 회귀 방지)

**예상 원인 가설:**

**사용자 원문:**

**분류:** UI 기능 추가

**관련 코드/기록:**

- Quick 260519-c08 hotfix 패턴: FAI 노드 / 측정 노드 / 결과 행 별 ROI 하이라이트
- Phase 37 hotfix A (1c11c35): Measurement 노드 이미지/ROI 해석을 측정 객체 참조로 (FindFAIContainingMeasurement)
- 현재 검사 트리거는 sequence 단위 (`Action_FAIMeasurement.Run`) — FAI 단위 분리 없음

**예상 작업 옵션:**

**Anti-goals:**
- Sequence 흐름 변경 최소화 (Phase 37 lenient + Phase 39 gate 회귀 0)
