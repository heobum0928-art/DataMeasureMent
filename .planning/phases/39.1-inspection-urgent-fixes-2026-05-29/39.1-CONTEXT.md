### G2: Item #1 CircleDiameter 파라미터 노출

- **D-G2-01**: **4 필드 PropertyGrid 노출** — `Circle_PolarStepDeg` (double, default 10.0) + `Circle_RectL1Ratio` (double, default 0.02) + `Circle_RectL2Ratio` (double, default 0.02) + `Circle_PolarEdgeSelection` (string, default "First"). 인스턴스 필드화 (현재는 `EdgeOptionLists` 정적 상수 직접 참조). *Why:* 사용자 원문 = "polar step + Length1 + Length2 누락" → EdgeSelection 까지 묶어 strip-loop 의 3 필수 항목 (feedback_halcon_measurepos_must_haves) 모두 사용자 제어 가능.
- **D-G2-02**: **L1/L2 = Ratio (0.0~1.0, 원 지름 대비)** — 현 `FaiCircleRectL1Ratio=0.02` 일치, INI byte-identical. Ratio 채택 이유: 일관 polar strip 크기가 원 지름에 자동 비례 → 큰 원/작은 원 동일 동작.
- **D-G2-03**: **INI 하위호환 = sentinel + EnsurePerRoiDefaults idempotent migration** — Phase 13 D-PRP-LOOP / Phase 22 IMG-01 패턴 차용. 신규 필드 default = sentinel (0.0 / ""), `FAIConfig.EnsurePerRoiDefaults` 또는 `CircleDiameterMeasurement` ctor 내 idempotent 채움. 기존 INI 로드 시 default 값 = `EdgeOptionLists.FaiCircle*` 상수와 일치 (10.0/0.02/0.02/"First").
- **D-G2-04**: **ICustomTypeDescriptor — `Circle_RadialDirection` 설정 시만 polar 4 필드 노출** — fit 경로 (RadialDirection=="") 에서는 polar 필드 hide. `FAIConfig.GetProperties` 동적 노출 패턴 (Phase 19 QUAL-03) 또는 `DynamicPropertyHelper.FilterProperties` 위임 채용. *Why:* fit 경로 사용자가 polar 필드 보고 혼원 방지, Phase 28 D-03 동등성 가드 강화.
- **D-G2-05 (anti-goal)**: **Datum CTH 동작 변경 0** — `DatumConfig` 와 `DatumFindingService.TryTeachCircleTwoHorizontal` 의 polar 경로는 그대로 `EdgeOptionLists.FaiCircle*` 정적 상수 사용. CircleDiameter 신규 인스턴스 필드는 FAI 측정 경로 **에만** 영향. Datum CTH 의 INI/결과 회귀 0.

### G3: Item #2 EdgeToLineDistance 수정 방향

- **D-G3-01**: **근본 원인 진단 lock = `DatumAngleRad+90°` 가정 파괴** — `EdgeToLineDistanceMeasurement.cs` L156-162 `measureX` 분기가 datum 1차 수평선 각도 + 90° 로 가상 수직선 구성. 실제 `DatumConfig.DetectedRefAngle2` (수직 기준선 각도) 와 다를 때 (부품이 정확한 직각 아닐 때) 정사영 결과 오차 → 수직선 못 미침/넘어감. **단, 사용자 보고 시나리오 실측 재현은 Plan 단계 first task.**
- **D-G3-02**: **수정 범위 = `measureX` 분기만 `DatumAngle2Rad` 사용** — `measureY` (수평선까지 거리) 분기는 `DatumAngleRad` 유지 (회귀 0). `measureX` 만 `DatumAngle2Rad` 로 교체. `axisR1/C1, axisR2/C2` 정의는 신규 angle 기반 재구성 (cos/sin 직접 사용).
- **D-G3-03**: **수정 위치 = 인라인 (현 `EdgeToLineDistanceMeasurement.TryExecute` 내부)** — Phase 31 D-04 `ComputeProjectionDistance` 헬퍼 통합 미채택. *Why:* Phase 31 Task 3(C) 보류 결정 (overlay footRow/footCol 회귀 방지) 유지. 인라인 수정으로 함수 1개 단수 변경 + overlay 재사용 보장.
- **D-G3-04**: **`DatumAngle2Rad==0` 시 폴백 = 현재 로직 (DatumAngleRad+90°)** — 주입 미수행 케이스 (TLI Datum 등 수직 기준선 명시 없는 알고리즘) 회귀 0 보장. CTH/VTH Datum 사용 시에만 실제 변경 적용. *Why:* `DetectedRefAngle2` sentinel 0 인 경우는 datum 자체에 2차 기준선 없는 상황 → 강제 적용하면 결과 0 또는 NaN 위험.
