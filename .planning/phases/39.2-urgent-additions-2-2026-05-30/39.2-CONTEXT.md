## D-G1 — Bottom E5 DualImage FAI Measurement (Item #1)

**알고리즘 컨셉 (사용자 명시 2026-05-30):**
- 양 ROI (각각 다른 이미지) 모두 strip-loop + edge detection + fit_line (EdgeToLineDistance 패턴 동형)
- **ROI A (점 형태):** fit_line 후 → 라인 **중점(midpoint)** 1점 선택 → "Point"
- **ROI B (라인 형태):** fit_line 후 → 시작점/끝점 = 라인 자체 → "Line"
- **거리:** `projection_pl(point → line)` 수직 정사영 거리 (EdgeToLineDistance Phase 23.1 / l5e-03 패턴 재사용)

**D-G1-01: 신규 Measurement 타입 — `DualImageEdgeDistanceMeasurement`**
- MeasurementBase 서브타입 (직접 상속, EdgeToLineDistance subclass 아님 — 다른 image source 흐름)
- TypeName = "DualImageEdgeDistance"

**D-G1-02: 점 선택 방식 — `midpoint` 고정**
- fit_line 결과 시작점 (s) / 끝점 (e) → midpoint = ((s+e)/2)
- 사용자 결정 0 (PointSelection enum 도입 X, 추후 phase 확장 후보)

**D-G1-03: ROI 슬롯 ↔ 이미지 컨벤션 — Claude 기본값 (사용자 추가 결정 시 변경 가능):**

- `TeachingImagePath_Vertical` (2차) → ROI B = "LineROI" (라인 형태 ROI)
- 슬롯 명시 컨벤션 (Phase 37 VTH-DualImage 패턴 재사용, 사용자 swap UI 미도입 — 단순성 우선)
- 향후 사용자 swap 필요 시 Phase 39.3 또는 hotfix 로 확장

**D-G1-04: Edge 검출 파라미터 — per-ROI 독립**

- LineROI_Row/Col/Phi/Length1/Length2
