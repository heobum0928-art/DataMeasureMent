# Phase 39.3 CONTEXT — DualImage FAI UX 재설계 (Bottom E5 carry-over)

## Goal
Phase 39.2 D-G1 (`DualImageEdgeDistanceMeasurement`) UAT FAIL 후속 — **측정 알고리즘은 유지**, UI/UX 를 Side Datum DualImage (Phase 34.1 / 37) 패턴으로 재설계.

## Root Cause Analysis (Phase 39.2 UAT FAIL)
- **`InspectionListView.xaml.cs:496-503`** `isRectRoiType` 가 7 Measurement 타입만 명시 (EdgeToLineDistance / EdgeToLineAngle / ArcEdgeDistance / ArcLineIntersectDistance / CompoundAngle / CompoundCenterC / CompoundCenterB / CompoundShortAxis). `DualImageEdgeDistanceMeasurement` 누락 → `btn_rectRoi.IsEnabled = false` → RectROI 드로잉 비활성.
- 이 측정 타입은 **2 ROI** (PointROI + LineROI) + **2 이미지** (RuntimeImageA + RuntimeImageB) 보유. 단순히 활성화만으로는 부족 — 가로/세로 이미지 슬롯과 ROI 가 연동되어 그릴 대상이 결정되어야 함.

## 6 Locked Decisions (D-G1~D-G5)

### D-G1: RectROI 드로잉 활성화 + ROI 선택 방식
**결정:** 단일 RectROI 드로잉 버튼 + **현재 표시 중인 이미지 슬롯에 따라 대상 ROI 자동 결정** (G2 swap 와 연동).
- PointROI 슬롯 표시 중 → PointROI_Row/Col/Phi/Length1/Length2 드로잉
- LineROI 슬롯 표시 중 → LineROI_Row/Col/Phi/Length1/Length2 드로잉
- 사용자 인지 부담 0 — "이미지 보면서 그 이미지의 ROI 만 그리면 됨"

**Why:** 사용자 명시 — "이미지를 보면서 해야 하지 않나". 별도 디스팅 드로잉 모드 / 2 버튼 UI 도입 0.

**구현 영향:**
- `InspectionListView.xaml.cs:496-503` `isRectRoiType` 에 `|| item.Param is DualImageEdgeDistanceMeasurement` 추가 (1 줄)
- MainView 의 RectROI 드로잉 → write-back 분기에 DualImage 타입 대응 (현재 활성 슬롯 상태 + 대상 ROI 결정 로직 추가)
