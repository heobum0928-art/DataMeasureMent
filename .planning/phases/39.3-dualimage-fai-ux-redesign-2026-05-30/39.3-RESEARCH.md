## 개요

Phase 39.3 은 **순수 UI/UX phase**. 측정 알고리즘은 39.2-01 baseline 그대로 (D-G4) 보존하고, `DualImageEdgeDistanceMeasurement` 의 PropertyGrid 입력 + RectROI 드로잉 + 가로/세로 이미지 swap UX 를 Phase 34.1 / 37 의 **Side Datum DualImage** 패턴과 동일하게 만든다. 사용자 인식 부담 0 — Datum DualImage 와 동작이 byte-identical 한 UX. 분기 추가는 정확히 **4 지점** (InspectionListView 1, MainView 3) + Measurement 본체에 `[InputFilePath]` attribute 1줄. 총 코드 추가량 매우 작지만, write-back 시 "현재 표시 중인 슬롯" 상태가 PointROI 와 LineROI 의 라우팅 키이므로 **상태 모델 통합** (Datum 의 `_selectedDatumForSwap` 를 MeasurementBase 로 확장 vs 별도 필드) 이 본 phase 의 단일 설계 결정점이다.

**Primary recommendation:** Datum 측 swap 인프라 (`btn_swapHorizontal/Vertical`, `_currentImageSource`, `UpdateImageSourceBadge`, `BadgeBrushHorizontal/Vertical`) 를 **재사용** (별도 버튼/배지 X). `_selectedDatumForSwap` 와 대칭하는 **`_selectedDualImageMeasurement`** 신규 필드 1개 추가. 활성화 분기 = "DualImage Measurement 선택 시 = `btn_swapHorizontal/Vertical.Visibility = Visible`, DualImage 외 = 기존 Datum 분기에 위임". swap 핸들러는 Datum 인지 Measurement 인지 분기 — 우선순위 = Measurement (트리 노드 selection 기준).
