# Phase 39.4 CONTEXT (SEED) — Bottom DualImage 수동 Swap UX 재설계

## Origin

Phase 39.3 (PARTIAL_SIGNED_OFF, 2026-05-30) Test 2 (Swap UX) PASS 후 사용자 발견:
- **Shot 이미지는 공통 자원** — 동일 Shot 의 다른 FAI 와 공유. 그러나 Phase 39.3 의 Horizontal swap 분기는 ShotConfig 이미지를 DualImageEdgeDistanceMeasurement 의 "가로축 티칭 이미지" 로 단독 점유.
- 결과: 작업자가 "이 이미지가 이 FAI 의 가로축 티칭 이미지인지, 다른 FAI 의 공유 이미지인지" 즉각 인지 불가.
- 실 카메라 grab 환경 (= POC 시연 기준) 에서 혼동 심화 예상.

## Goal

DualImage 가로/세로 **양측 모두 Measurement 단위 명시 경로** 로 재설계. Datum DualImage (Phase 22 IMG-01 + Phase 37 RT-2) 패턴 차용 — 가로/세로 모두 TeachingImagePath_* 명시.

## Carry-Over (CO-39.3-01)
