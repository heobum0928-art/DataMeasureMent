<decisions>
## Implementation Decisions

### #2 — 이미지 뷰어 overlay On/Off 토글 (2개 분리, 기본 ON)
- **2개 분리 토글** (사용자 결정): (A) "측정 overlay" 토글 — FAI 측정 결과 overlay(`EdgeInspectionOverlay`: FAI-Edge*, FAI-DistLine 등, `SetInspectionOverlays` 경로). (B) "Datum 라인" 토글 — Datum 검출 수평/수직 라인 + 검출 십자/origin (`HalconDisplayService.RenderDatumOverlay` 경로).
- **기본 상태 = 둘 다 ON** (기존 동작 유지). 체크 해제 시 해당 overlay 군만 미표시.
- **컨트롤 위치:** MainView 이미지 뷰어 영역(HalconViewerControl 인근)에 체크박스 2개. 기존 MainView UI 스타일 답습.
- **게이팅 동작:** 토글 OFF 시 해당 overlay 세트 렌더를 건너뛴다. 라이브 검사 결과 렌더 경로와 노드 클릭 재현 경로(`RenderInspectionResultForNode`/`HighlightSelectedRoi`/`RenderDatumOverlay`) **둘 다**에 적용되어야 한다. 토글 변경 시 현재 화면 즉시 갱신(재렌더).
- **상태 보존:** 세션 내 메모리 플래그로 충분 (INI 영속화 불요 — Claude 재량, 단순 우선).

### #3 — 트리 기본 Shot 레벨 접기 + 펼치기
- **기본 펼침 = Shot 레벨까지만.** Sequence > Shot 노드는 보이고, 그 하위 FAI/Measurement 는 접힘.
- 사용자가 Shot 노드를 펼치면 FAI/Measurement 상세 표시 (표준 TreeView 펼치기).
- 현재 `InspectionListView.ListView_Loaded` 의 전체 auto-expand 동작을 **Shot 레벨 제한**으로 변경. (auto-expand 전체 → Shot 까지만)
- **회귀 가드:** 노드 선택 기반 ROI 표시/하이라이트/PropertyGrid 전환은 펼침 상태와 무관하게 동작해야 한다 (접힌 하위 노드 선택은 사용자가 펼친 뒤 수행 — 기존 선택 로직 무변경).

### #4 — Polygon ROI UI 숨김 (데이터 호환 유지)
- Polygon ROI 를 **UI에서 숨김**: ROI 타입 선택/생성 버튼/메뉴 등 사용자 진입점에서 Polygon 제외.
- **데이터 호환 유지:** `RoiShape.Polygon` enum 값과 `HalconDisplayService` 의 Polygon 렌더 분기는 **보존** — 기존 INI 레시피에 Polygon ROI 가 있어도 로드/렌더가 깨지지 않아야 한다. 신규 생성·노출만 차단.
- 완전 제거 아님 (anti-goal).

### Claude's Discretion
- 토글 체크박스의 정확한 배치/스타일/라벨 문구.
- 토글 상태 메모리 보관 방식 (MainView 필드 vs ViewModel).
- 트리 펼치기 affordance (펼치기/접기 버튼 추가 여부 — 선택).
- Polygon 숨김 구현 위치 (ROI 타입 enum 필터 / 버튼 Visibility / 생성 분기 가드 중 최소 침습 선택).
</decisions>
