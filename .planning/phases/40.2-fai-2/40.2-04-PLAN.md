    **Test 1 — 폴더 구조 + 파일 쌍 생성**
    1. `WPF_Example\bin\x64\Debug\Result\Image\{오늘날짜 예 260610}\{검사시각 HHmm 예 1530}\` 폴더로 이동.
    2. `original\` 과 `capture\` 두 하위 폴더가 존재하는지 확인.
    3. 각 폴더에 검사한 Shot 의 FAI 개수만큼 PNG 가 있는지 확인 (FAI N개 → original N장, capture N장).
    예상: 두 폴더 존재 + FAI별 1장씩.

    **Test 2 — 파일명 규칙 + origin/capture 쌍**
    1. original 폴더 파일명: `origin_{시퀀스}_{FAI}_{측정점}_{HHMMSSfff}.png` (예 `origin_Top_FAI_A1_P1_153012345.png`). 측정점 1개면 P1, 2개면 P1P2, 없으면 segment 생략.
    2. capture 폴더에 동일 시퀀스/FAI/측정점/timestamp 의 `capture_...png` 가 1:1 로 존재 (prefix 만 다름).
    예상: origin_/capture_ 가 동일 timestamp·segment 로 쌍을 이룸.

    **Test 3 — 캡쳐 이미지에 오버레이, 원본은 원본만**
    1. `capture_...png` 를 더블클릭으로 열기 → 측정 오버레이(녹/적 에지선, 청록 측정선, OK/NG 등)가 이미지 위에 그려져 있는지 확인.
    2. 같은 쌍의 `origin_...png` 를 열기 → 오버레이 없이 원본 이미지만 보이는지 확인.
    예상: capture=오버레이 있음, origin=오버레이 없음.

    **Test 4 — 엑셀 export 파일명 2컬럼**
    1. 리뷰어에서 방금 검사 cycle 선택 → [엑셀 export] 버튼 클릭 → xlsx 저장.
    2. xlsx 를 열어 테이블 헤더(5행)에 `원본이미지 파일명`, `캡쳐이미지 파일명` 2컬럼이 있고 기존 `이미지` 컬럼이 없는지 확인.
    3. 두 컬럼에 파일명(경로 아님)이 채워졌는지 + 셀이 하이퍼링크가 아닌 일반 텍스트인지 확인.
    예상: 10컬럼, 파일명 텍스트, 하이퍼링크 없음.

    **Test 5 — throughput (선택)**
    1. 검사를 연속 수 회 실행 → 앱이 멈춤/지연 없이 동작하고 PNG 가 누락 없이 쌓이는지 확인.
    예상: 비동기 저장으로 검사 흐름 영향 미미.
  </how-to-verify>
