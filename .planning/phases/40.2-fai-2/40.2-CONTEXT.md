## Phase Boundary

Phase 40 OUT-02(1회 검사 엑셀 export) UAT 도중 발견된 후속 요구. 현재 엑셀의 "이미지" 컬럼은 원본 이미지로의 하이퍼링크(커밋 3385f7c)뿐이며, 측정 오버레이(측정선·에지·OK/NG)는 디스크에 이미지로 저장되지 않고 벡터 데이터(`LastOverlays`)로만 보관되어 리뷰어가 열 때마다 실시간 재렌더된다.

이 phase는:
1. 검사 시점에 각 FAI에 대해 "원본 이미지"와 "측정 오버레이가 입혀진 캡쳐 이미지"를 각각 PNG 파일로 디스크에 저장한다.
2. 엑셀 export에서 하이퍼링크 컬럼을 제거하고, 두 파일명(원본/캡쳐)을 텍스트 컬럼으로 기입한다.

**In scope:** FAI별 캡쳐 이미지 생성·저장, DTO 필드 추가, 엑셀 컬럼 교체.
**Out of scope:** 동영상 임베드(Excel 비지원), 이미지를 xlsx 내부에 바이너리 임베드(파일명 텍스트만), Bottom/Side 외 시퀀스 일반화(우선 기존 검사 시퀀스 경로 따름).
</domain>

<decisions>
## Implementation Decisions

### 캡쳐 이미지 단위
- **FAI별 1장**. 해당 FAI의 측정 오버레이만 렌더(샷 전체 합산 아님). 한 샷에 FAI가 N개면 캡쳐 이미지도 N장.

### 저장 폴더 구조
- 날짜\시각 폴더 아래 original/capture 분리:
  - `Result\Image\{YYMMDD}\{HHMM}\original\` — 원본 이미지
  - `Result\Image\{YYMMDD}\{HHMM}\capture\` — 측정 오버레이 캡쳐 이미지
- `Result` 루트 위치는 기존 결과 저장 경로 규칙을 따른다(RawImageSaveService / SystemSetting 결과 경로 확인하여 정합).

### 파일명 규칙
- 원본: `origin_{시퀀스}_{FAI}_{측정점}_{HHMMSSfff}.png`
- 캡쳐: `capture_{시퀀스}_{FAI}_{측정점}_{HHMMSSfff}.png`
- `{시퀀스}` 예: `Top`
- `{측정점}` 가변: 해당 FAI에 실제 존재하는 측정점을 이어붙임(P1만 있으면 `P1`, P1·P2 둘 다면 `P1P2`). 측정점이 없는 타입이면 생략 또는 빈 segment 처리(planner가 측정명/구조에서 점 식별).
- `{HHMMSSfff}` = 시분초밀리초.
- 원본/캡쳐는 동일한 timestamp·segment를 공유하여 쌍을 이룬다.

### 캡쳐 이미지 생성 방식
- 검사 시점에 HALCON 버퍼 윈도우(off-screen)에 원본 이미지 + 해당 FAI의 오버레이(벡터, `InspectionOverlays`/`LastOverlays`)를 렌더 → `dump_window`(또는 동등 API)로 HImage 취득 → PNG 저장.
- 리뷰어가 화면에 그리는 오버레이 렌더 로직(`ReviewerWindow`/`HalconViewerControl`/`HalconDisplayService`)을 헤드리스로 재사용/참조.
- 시퀀스 스레드(UI 없음)에서 동작해야 하므로 UI 컨트롤 의존 없는 버퍼 윈도우 경로 사용.
- 기존 RawImageSaveService 비동기 워커 패턴을 참고해 검사 throughput에 영향 최소화(가능하면 비동기 저장).

### 엑셀 export 변경 (ExcelExportService.cs)
- 현재 D-07 "이미지" 하이퍼링크 컬럼(헤더 `이미지`, 커밋 3385f7c) **제거**.
- 대체: 텍스트 2컬럼 `원본이미지 파일명`, `캡쳐이미지 파일명`. 경로 아닌 **파일명만**(예: `capture_Top_FAI_A1_P1P2_153012345.png`). 하이퍼링크 없음.
- 헤더 배열/컬럼 인덱스 갱신(현재 9컬럼 → 원본/캡쳐 2컬럼으로 조정).

### DTO 변경
- `CycleResultDto`/`ShotResultDto`/`FaiResultDto` 계열(`UI/ViewModel/CycleResultDto.cs`)에 원본 파일명·캡쳐 파일명 필드 추가. 현재는 `ResultImagePath`(=원본 전체 경로)만 존재.
- 파일명 단위는 FAI별이므로 FAI 레벨 DTO에 필드가 자연스러움(또는 Shot 레벨 + 측정점). planner가 FAI별 캡쳐 단위에 맞게 배치.
- `CycleResultSerializer.cs`에서 채움.

### Claude's Discretion
- 측정점 식별 정확한 방법(측정명 파싱 vs 측정 구조의 point 필드).
- 버퍼 윈도우 크기/스케일, PNG 압축 옵션.
- 파일명에서 측정점이 비는 경우의 segment 처리.
- 비동기 저장 vs 동기 저장(throughput 영향 측정 어려우면 단순 동기도 허용).
- original 이미지 저장: 기존 RawImageSaveService 산출물 재사용 가능하면 재사용, 아니면 별도 저장.
</decisions>
