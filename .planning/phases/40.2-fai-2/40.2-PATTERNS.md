## Pattern Assignments

### 1. `CaptureImageSaveService.cs` (신규, service / event-driven + file-I/O)

**Analog:** `WPF_Example/Utility/RawImageSaveService.cs` (전체 116줄 — 그대로 복제할 골격)

이 서비스는 RawImageSaveService를 **거의 그대로 복제**하되, request DTO에 `FaiName`/`Sequence`/`MeasurePointSegment`/`IsCapture`(또는 original/capture 2장) 필드를 추가하고 파일명 규칙만 교체한다.

**복제할 비동기 워커 골격** (RawImageSaveService.cs:24-69) — ConcurrentQueue + AutoResetEvent + volatile 종료 플래그:
```csharp
private readonly ConcurrentQueue<RawImageSaveRequest> _queue = new ConcurrentQueue<RawImageSaveRequest>();
private readonly AutoResetEvent _signal = new AutoResetEvent(false);
private readonly Thread _workerThread;
private volatile bool _isStopping;
private volatile bool _isStarted;
// 생성자: ThreadPriority.BelowNormal, IsBackground=true (검사 throughput 보호)
// WorkLoop: TryDequeue → SaveRequest → 없으면 _signal.WaitOne(100). 종료 시 잔여 큐 flush.
```

**복제할 request DTO + Dispose 계약** (RawImageSaveService.cs:10-22) — HImage 소유권을 request가 갖고 저장 후 Dispose:
```csharp
public sealed class RawImageSaveRequest : IDisposable {
    public HImage Image { get; set; }
    public DateTime Timestamp { get; set; } = DateTime.Now;
    public void Dispose() { Image?.Dispose(); Image = null; }
}
```

**복제할 저장 + 파일명 sanitize 패턴** (RawImageSaveService.cs:71-105). 단, 경로/파일명만 교체:
```csharp
// 기존(RawImageSaveService): GetLogSavePath(ELogType.Image,"Raw",yyyyMMdd) + "{HHmmssfff}_{seq}_{act}_{id}.png"
// 신규(이 phase): 아래 "결과 경로" 패턴 + "origin_{시퀀스}_{FAI}_{측정점}_{HHMMSSfff}.png" / "capture_..."
request.Image.WriteImage("png", 0, filePath);   // HImage PNG 저장 API (RawImageSaveService.cs:85 동일)
```
> `WriteImage("png", 0, filePath)`가 이 코드베이스의 표준 HImage→PNG 저장 호출이다 (RawImageSaveService.cs:85). `dump_window_image`로 얻은 캡쳐 HImage도 동일 호출로 저장한다.

**enqueue 게이트 + 소유권 복사 패턴** (Action_TopInspection.cs:283-296의 `QueueRawImageSave`) — saver null 가드 + `image.CopyImage()`로 소유권 분리:
```csharp
if (image == null || SystemHandler.Handle.CaptureImageSaver == null) return;
SystemHandler.Handle.CaptureImageSaver.Enqueue(new CaptureImageSaveRequest {
    Image = image.CopyImage(),   // 워커가 dispose할 사본 — 호출 스레드 원본 보호
    ...
});
```

**SystemHandler 등록** — `SystemHandler.RawImageSaver`와 동일하게 싱글턴 필드 추가 + `Initialize()`에서 `.Start()`, `Release()`에서 `.Dispose()`. (RawImageSaver 등록 지점을 grep하여 미러링: `Grep("RawImageSaver", path:"WPF_Example/SystemHandler.cs")`)

### 2. `OverlayCaptureRenderer.cs` (신규, headless 렌더 → dump → PNG) — **핵심 신규 패턴, analog 부분만 존재**

**Analog (렌더 로직):** `WPF_Example/Halcon/Display/HalconDisplayService.cs` `Render()` (19-282) — 이미 `HWindow + HImage + IEnumerable<EdgeInspectionOverlay>`를 인자로 받아 오버레이를 그린다. 이 메서드를 **그대로 재사용**한다(UI 컨트롤 의존 없음, `HWindow`만 받음).

**Analog (재렌더 호출 순서):** `ReviewerWindow.DisplayCycle` (ReviewerWindow.xaml.cs:155-171) — 헤드리스에서도 동일 순서를 지켜야 함:
> **순서 계약 (RESEARCH Pitfall 6):** `LoadImage(이미지) → SetInspectionOverlays(오버레이)`. HalconDisplayService.Render는 image를 먼저 `DispObj`한 뒤(L41) 오버레이를 그린다(L137~). 헤드리스도 `Render(bufferWindow, image, rois:null, selectedRoiId:null, inspectionOverlays: faiOverlays)` 한 번 호출이면 동일 결과.

**신규 = 버퍼(off-screen) 윈도우 생성 + dump.** 코드베이스에 기존 예시 없음. HALCON 24.11 표준 API로 신규 작성:
```csharp
// 이미지 크기로 buffer 윈도우 open (UI HWND 불필요 — 시퀀스 스레드 안전)
image.GetImageSize(out HTuple w, out HTuple h);
HTuple win;
HOperatorSet.OpenWindow(0, 0, w, h, 0, "buffer", "", out win);   // "buffer" = off-screen
HWindow hwin = new HWindow(win.IP);   // HalconDisplayService.Render 가 HWindow 타입을 요구
try {
    HOperatorSet.SetPart(win, 0, 0, h - 1, w - 1);   // 전체 이미지 매핑
    _displayService.Render(hwin, image, null, null, null, faiOverlays, null);  // 기존 렌더 재사용
    HObject dumped;
    HOperatorSet.DumpWindowImage(out dumped, win);   // 윈도우 → HImage (검증: dump_window_image.html 존재 확인)
    var captureHImage = new HImage(dumped);
    // captureHImage 를 CaptureImageSaveService 로 enqueue (소유권 이전, CopyImage 불필요 — 이미 새 객체)
}
finally {
    HOperatorSet.CloseWindow(win);   // 버퍼 윈도우 누수 방지 (필수)
}
```
> **검증됨:** `dump_window_image` (= `HWindow.DumpWindowImage(out HImage)`) 와 `dump_window` 둘 다 HALCON 24.11 설치본에 존재. `"buffer"` 모드 OpenWindow는 화면 출력 없는 off-screen 렌더 표준.

**존중할 HOperatorSet try/catch 컨벤션** (CLAUDE.md + HalconDisplayService 전역):
> 모든 `HOperatorSet.*` 렌더 블록은 `try { } catch { /* suppress display errors */ }`. CloseWindow는 반드시 `finally`. 렌더 실패가 검사 스레드를 죽이면 안 됨 → 캡쳐 실패 시 PNG만 누락, 검사 계속 (RawImageSaveService.SaveRequest:87-92의 catch+Logging 패턴 동일 적용).

**측정점(P1/P2) segment 식별 — planner 필독:**
> 이 코드베이스에는 명시적 `P1`/`P2`/`Point1` 필드가 **없다**. 측정점은 `FaiResultDto.LastOverlays` → 각 `EdgeInspectionOverlay.Points` (`List<EdgeInspectionPoint>{Row,Col}`, EdgeInspectionOverlay.cs:6-26)에서만 나온다.
> - `EdgeInspectionOverlay.RoiId`가 `"FAI-Edge..."`로 시작하는 오버레이가 검출 에지점(`Points`)을 담는다 (HalconDisplayService.cs:217, Action_FAIMeasurement.cs:280).
> - 한 FAI의 측정 오버레이 중 `Points`가 있는 검출점의 **개수**로 `{측정점}` segment 구성 권고: 점 1개 → `P1`, 2개 → `P1P2`. 점 없는 타입(각도 등)은 빈 segment.
> - **대안(권고):** FAI 하위 `Measurements` 리스트를 순회해 측정 개수/타입으로 segment를 만들 수도 있으나, CONTEXT는 "FAI별 1장 = 해당 FAI 오버레이만 렌더"이므로 오버레이 점 카운트가 가장 직접적이다. 측정 구조보다 **오버레이 Points 카운트**를 1순위로.

### 4. `CycleResultSerializer.cs` (수정, service / DTO 빌드)

**Analog:** 자기 자신 — `BuildDto` FAI 루프 (CycleResultSerializer.cs:78-124).

## Shared Patterns

### A. 비동기 저장 워커 (ConcurrentQueue + volatile + AutoResetEvent)
**Source:** `WPF_Example/Utility/RawImageSaveService.cs:24-114`
**Apply to:** `CaptureImageSaveService` (신규)
- `volatile bool _isStopping/_isStarted`, `ConcurrentQueue<T>`, `AutoResetEvent`, `IsBackground` BelowNormal 스레드.
- 종료 시 잔여 큐 flush (L66-68). `Dispose`에서 `_signal.Set()` + `Join(1000)`.

### B. HImage 소유권 & Dispose
**Source:** `RawImageSaveRequest.Dispose` (RawImageSaveService.cs:18-21), `QueueRawImageSave` (Action_TopInspection.cs:290 `image.CopyImage()`), Action_FAIMeasurement.cs:170 `using(var image=...)`, :251-255 try/finally Dispose.
**Apply to:** 캡쳐 서비스 + Action_FAIMeasurement 삽입부 + OverlayCaptureRenderer
- 호출 스레드 원본은 `CopyImage()` 사본만 워커로 넘긴다. 워커가 저장 후 `request.Dispose()` (finally, RawImageSaveService.cs:90-92).
- 버퍼 윈도우는 `HOperatorSet.CloseWindow`를 finally에서 보장.

### C. HOperatorSet try/catch swallow (렌더 전용)
