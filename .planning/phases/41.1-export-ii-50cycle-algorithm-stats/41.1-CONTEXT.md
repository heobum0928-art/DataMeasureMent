## Phase Boundary

N회(예: 100회) 반복 측정을 **오토시퀀스**로 자동 실행하고, 회차별 원본(raw) 측정값을 CSV 매트릭스(행=회차, 열=검사항목)로 남긴다. Offline(SIMUL) 모드에서도 회차별로 미리 준비된 이미지 폴더를 참조해 동일한 오토시퀀스가 동작해야 한다(반복 촬영 이미지가 부족한 지금 상황에서 반복성 로직을 검증하는 유일한 경로).

**중요한 스코프 정정 (코드 정찰 결과):** 이 phase는 2026-06-16에 "실행 0건"으로 DEFERRED됐지만, 실제로는 이미 상당 부분 코드가 존재한다 (Quick task로 만들어짐, 정식 GSD 계획/서명 없이):
- `RepeatRunService.StartFromImages()` — 폴더 내 이미지 파일들을 순서대로 자동 반복 실행 (오토시퀀스 이미 구현됨)
- `RepeatExcelExportService.Export()` — 2-시트 xlsx (반복도 통계 mean/stddev/range/Cpk + 알고리즘별 통계) — OUT-03/04 요약 통계는 이미 있음
- `ReviewerWindow`에 "이미지 폴더 반복 검사" + "반복도 엑셀 export" 버튼 이미 연결됨

**실제 남은 갭 (이번 phase 스코프):**
1. **회차별 raw CSV 매트릭스 export** — 기존 xlsx는 요약 통계만 냄. 회차 1~N을 행으로, 검사항목(Shot_FAI_측정명)을 열로 하는 원본값 매트릭스가 없음 → 신규 추가.
2. **멀티샷 오프라인 반복 폴더 구조** — 현재 `ApplyCurrentImage()`는 폴더 안 이미지 1장을 **모든 Shot에 동일하게** 적용. 실제 레시피는 시퀀스당 Shot이 여러 개(Z축 위치별)이며 각 Shot은 자기 `SimulImagePath`를 따로 가짐 (`ShotConfig`/`Action_FAIMeasurement.cs:189` 참조) — 회차마다 Shot별로 다른 이미지를 쓸 수 있어야 정확한 반복성 검증이 됨. 현재 구조로는 불가능 → 회차별 하위폴더 + ShotName 매칭 구조로 확장 필요.

**Out of scope (변경 없음, REQUIREMENTS.md/PROJECT.md 명시):**
- 정식 Gage R&R ANOVA (operator × part × trial)
- 기존 xlsx 요약 통계(mean/stddev/range/Cpk, 알고리즘별 통계) 로직 변경 — 그대로 유지, 신규 CSV는 별도 산출물로 추가

<decisions>
## Implementation Decisions

### 오프라인 반복 폴더 구조 (D-01~D-03)
- **D-01:** 회차별 **하위 폴더** 방식 채택. 상위 폴더 선택 시 그 밑에 `1/`, `2/`, ... `N/` 폴더가 있고, 각 폴더가 1회차에 대응한다.
- **D-02:** 각 회차 폴더 안의 이미지 파일은 **ShotName과 파일명을 매칭**하여 해당 Shot에 적용한다 (예: `Top_Shot1.bmp` → ShotName="Shot1"인 Top 시퀀스 Shot에 적용). 매칭되는 파일이 없는 Shot은 해당 회차에서 스킵(기존 "SimulImagePath 없음 → 해당 Shot만 skip" 정책과 동일 — `Action_FAIMeasurement.cs` 기존 동작 재사용).
- **D-03:** 순서(Shot 리스트 인덱스) 기준 매칭이 아니라 **이름 기준 매칭**을 채택한 이유: Shot 순서가 바뀌어도 안전. `RepeatRunService.ApplyCurrentImage()`를 이 매칭 로직으로 교체(기존 "폴더 1개 + 파일 N장 + 전체 Shot 동일 적용" 모드는 대체됨 — 하위 호환 유지 여부는 Claude's Discretion 참고).

### 회차별 raw CSV 매트릭스 (D-04~D-06)
- **D-04:** 행 = 회차 번호(1~N, `_repeatCycles`(List&lt;CycleResultDto&gt;) 순서 기준 인덱스+1). 열 = 검사항목, 열 키는 `{ShotName}_{FAIName}_{MeasurementName}` 조합.
- **D-05:** 셀 값 = **측정값(`MeasurementResultDto.LastMeasuredValue`)만**. 판정(OK/NG)은 이 CSV에 포함하지 않는다 — 통계 분석(Gage R&R 반복성 계산)에 바로 쓸 수 있는 순수 숫자 매트릭스가 목적. 기존 xlsx 요약 통계 시트가 판정/불량 집계를 이미 담당하므로 중복 없음.
- **D-06:** 미측정/DATUM_FAIL 등으로 값이 없는 셀은 빈 값 또는 명시적 마커(예: 빈 문자열)로 남긴다 — 구체 표기는 Claude's Discretion(기존 `RepeatMeasurementStats`의 skip 판별 로직 참고).

### UI 진입점 (D-07)
- **D-07:** `ReviewerWindow`의 기존 "이미지 폴더 반복 검사" / "반복도 엑셀 export" 버튼 옆에 **"회차별 CSV export" 버튼을 나란히 추가**한다. 기존 흐름(반복 실행 → `_repeatCycles` 누적 → 여러 export 옵션 중 선택) 유지. 자동 생성 방식은 채택하지 않음 — 사용자가 필요할 때만 export.

### 검증 범위 (D-08)
- **D-08:** 실제로 확보된 반복 이미지는 **회차당 1장뿐**(멀티샷 각각의 이미지는 없음). 이번 phase의 실행단계 검증(dry-run)은 **그 1장을 회차 폴더 안 모든 Shot에 동일하게 복사배치**해서(사실상 기존 단일이미지 모드와 동일한 입력으로) 폴더 구조 인식 + 매칭 로직 + CSV 생성까지 **파이프라인이 정상 동작하는지만** 확인한다. Shot별로 실제 다른 이미지를 사용한 반복성 차이 검증은 **장비/이미지 확보 후로 이연**한다.
- 이번 phase 동안 discuss/plan/execute 전 구간에서 **여러 에이전트(코드 리뷰/로직 검증)를 적극 운용**하여 실행 검증이 제한적인 만큼 코드 신뢰도를 높인다 (사용자 명시 요청).

### Claude's Discretion
- 기존 "폴더 1개 + 파일 N장 + 전체 Shot 동일 적용" 단일이미지 모드의 하위 호환 유지 여부 (완전 대체 vs 두 모드 공존). 코드 중복 최소화 관점에서 결정.
- CSV 파일명/저장 경로 규칙 (`ResultSavePath` 등 기존 패턴 참고, SaveFileDialog 기본 파일명 등).
- CSV 인코딩(UTF-8 BOM — 한글 헤더 Excel 호환) 및 콤마/따옴표 이스케이프(RFC4180) 처리.
- 미측정 셀 표기 방식(빈 문자열 vs 마커).
- `RepeatRunService` 리팩토링 범위(기존 `_imagePaths`/`ApplyCurrentImage` 교체 시 `StartFromImages` 시그니처 변경 여부, `BatchRunService`와의 코드 공유 패턴 재확인).
