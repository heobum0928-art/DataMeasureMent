## 오프라인 반복 폴더 구조

| Option | Description | Selected |
|--------|-------------|----------|
| 회차별 하위폴더 + Shot이름 매칭 | 1,2,...N 하위폴더, 각 안에 ShotName 매칭 파일. 순서 무관, 안전 | ✓ |
| 회차별 하위폴더 + 파일 순서대로 매칭 | 이름 무관, recipeManager.Shots 순서 매칭. 단순하지만 순서변경 시 오적용 위험 | |

**Notes:** 코드 정찰 결과 실제 레시피는 시퀀스당 Shot이 여러 개(Z축 위치별)이고 각 Shot이 독립된 SimulImagePath를 갖는다는 걸 확인 후 제시. 현재 `RepeatRunService.ApplyCurrentImage()`는 전체 Shot에 동일 이미지를 적용하는 방식이라 멀티샷 반복성 검증에는 부적합함을 사용자에게 설명.

## 검증 범위 (dry-run)

**질문 1:** 지금 당장 검증용 반복 이미지가 몇 장이라도 있는지.
**User's answer:** "회차별로는 1장씩 밖에 없어" — 자유 응답. 멀티샷 각각의 이미지는 없고 회차당 1장뿐임을 확인.

**질문 2 (후속):** 회차당 1장뿐이라면, 그 1장을 회차 폴더 안 모든 Shot에 동일 복사배치해서 파이프라인(폴더 인식+매칭+CSV 생성)만 확인하는 dry-run으로 충분한지.

| Option | Description | Selected |
|--------|-------------|----------|
| 네, 그걸로 충분 | 1장 복사배치 스모크 테스트만, 실 Shot별 이미지 검증은 장비 확보 후 | ✓ |
| 불충분, 코드 리뷰까지만 | dry-run 자체를 하지 않고 코드 리뷰만으로 이번 phase 성공기준 삼음 | |

**User's choice:** 네, 그걸로 충분 (권장안)
**Notes:** 사용자 원 요청("우선 기능 만들고 discuss로 여러 에이전트 운용해서 검증, 추가검증은 나중에 장비 돌면")과 일치. 최종 마무리 질문에서 추가 논의 없이 CONTEXT.md 작성 진행으로 확정.
