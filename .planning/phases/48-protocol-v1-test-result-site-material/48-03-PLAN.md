<protocol_spec_byte_reference>
**Vision-Protocol-v1.0.md RESULT 포맷 (lines 44-48) — byte 정답지:**
- 정상(케이스1, 마지막·전체 OK): `$RESULT:site;P;count;id1=val1=OK,...@`
- NG(케이스2, 마지막 NG 있음 or Datum 실패): `$RESULT:site;F;count;id1=val1=NG,...@`
- Buffer(케이스3, Datum 샷): `$RESULT:site;B;count;id1=val1=OK,...@` 또는 빈 응답 `$RESULT:site;B;0;@`
- 구분자: 헤더(`;`) / 항목(`,`) / 항목내부(`=`).

**핵심 byte 구조 (STX/ETX 는 TcpServer Header/Trailer 가 부착 — 직렬화 문자열에는 미포함):**
```
RESULT:{site};{P|F|B};{count};{id1}={val1}={OK|NG},{id2}={val2}={OK|NG},...
```
- `site` 다음 첫 구분자 = `;` (헤더 구분자, **현행 `,` 아님**)
- `{P|F|B}` = cycle 종합 판정 1글자
- `count` = FAI 항목 개수 (FAICount)
- 마지막 `;` 뒤부터 FAI 항목들. 항목 간 = `,`. 항목 내부 = `=` (id=val=judge).
- val = 측정값 `0.000` 포맷 (DistanceMm.ToString("0.000")).
- Datum 샷(count=0): `RESULT:{site};B;0;` (마지막 `;` 뒤 항목 없음).
</protocol_spec_byte_reference>

// 260622 hbk Phase 48 PROTO-02: FAI 항목들을 id=val=judge,... 로 직렬화 (항목 간 ',').
private static string BuildFaiItemsV1(TestResultPacket testPacket)
{
    string szItems = "";
    int nCount = testPacket.FAICount;
    for (int i = 0; i < nCount; i++)
    {
        FAIResultData faiData = testPacket.FAIResults[i];
        bool bNeedsSeparator = i > 0;
        if (bNeedsSeparator)
        {
            szItems += MSG_RESULT_ITEM_SEP;          // ','
        }
        szItems += faiData.FAIName;                   // id
        szItems += MSG_RESULT_INNER_SEP;              // '='
        szItems += faiData.DistanceMm.ToString("0.000"); // val
        szItems += MSG_RESULT_INNER_SEP;              // '='
        szItems += MapFaiJudgement(faiData);          // OK|NG
    }
    return szItems;
}
