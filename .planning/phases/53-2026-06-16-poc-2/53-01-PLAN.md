3. 격자 간격 수집 (Pattern 2 — telecentric 축정렬 가정):
   - `List<double> CollectRowAdjacentColGaps(HTuple rows, HTuple cols, double pitchGuess)` 와 `CollectColAdjacentRowGaps(...)` private static 헬퍼.
   - pitchGuess: 모든 코너 bounding box 대각/sqrt(cornerCount) 근사 또는 1차 패스로 최근접 이웃 간격 중앙값. (간단히: 행 버킷팅용 허용오차 = pitchGuess * 0.5.)
   - 행 버킷팅: row 값을 round(row / pitchGuess) 인덱스로 버킷 → 같은 버킷 내 col 정렬 → 인접 col 차 수집(가로 간격 gapX). 동일하게 열 버킷팅 → row 차 수집(세로 간격 gapY).
   - **이상치 절사 필수(Anti-Pattern: mean 직접 사용 금지):** `private static double Median(List<double> vals)` 헬퍼 추가(정렬 후 중앙값). 2×피치 점프는 median 이 자연 배제.
4. mm/px 산출(D-02):
```csharp
double medGapX = Median(gapsX);
double medGapY = Median(gapsY);
double medGap = (medGapX + medGapY) / 2.0;
if (medGap <= 0) { error = "유효 간격 없음"; return false; }
double mmPerPixel = knownMmPerCell / medGap;
double mmPerPixelX = knownMmPerCell / medGapX;
double mmPerPixelY = knownMmPerCell / medGapY;
```
5. 중앙↔외곽 편차%(Pattern 3 — D-05 게이트): `private static double ComputeCenterOuterDeviationPct(HTuple rows, HTuple cols, HImage image, out double centerMean, out double outerMean)`:
   - 이미지 Width/Height (`image.GetImageSize(out w, out h)`) 로 중심(cx,cy), 반대각선 `diag = sqrt(w*w+h*h)/2`.
   - 각 인접 간격의 중점 거리 r. r <= diag*0.33 → 중앙부, r >= diag*0.66 → 외곽부.
   - `centerMean = mean(중앙부)`, `outerMean = mean(외곽부)`. 한쪽 그룹이 비면 devPct=0 처리(가드).
   - `devPct = Math.Abs(outerMean - centerMean) / centerMean * 100.0`.
6. 결과 채움:

result = new CalibrationResult {
