# 57.1-04 — Test Find 에 패턴 보정 연결 (티칭 단계 보정 육안 확인)

<objective>
티칭 단계의 "Test Find" 버튼(`BtnTestFindDatum_Click`)이 런타임 검사와 **동일한 패턴 보정 경로**(`InspectionSequence.TryComposeAlign`)를 타도록 연결한다. 이로써 사용자가 검사를 돌리지 않고도 Test Find 만으로 "패턴매칭 → ROI 가 부품을 따라 보정되는지" + cyan 보정 ROI + `[ALIGN]/[ALIGN-ROI]` 로그를 그 자리에서 육안 확인할 수 있다.

현 상태: Test Find 는 `svc.TryFindDatum()` 을 `AlignPreTransform=null`(보정 없음)로 직접 호출 → ROI 가 공칭 위치 고정, 패턴 보정 미반영.
</objective>

<background>
- `InspectionSequence.TryComposeAlign(datum, refImage[, refImageVertical], modelPath, out error)` 은 ① 패턴매칭(TryFindPose) ② θ(2-패턴 baseline 포함) ③ alignRigid 산출 ④ `detectSvc.AlignPreTransform=alignRigid` 로 datum 검출까지 수행하고, `datum.DetectedOrigin*` / `datum.CurrentTransform=alignRigid` / `datum.LastFindSucceeded=true` 를 write-back 한다 (InspectionSequence.cs:454~576). 이는 Test Find 의 `SetDatumOverlay(datum, true, ...)` 오버레이가 읽는 바로 그 값들이다.
- TryComposeAlign 은 sequence 실행 상태에 의존하지 않는다(모든 입력을 인자로 받음, 인스턴스 상태는 transient `_datumTransforms` 뿐 — 런타임 매 사이클 재계산되므로 티칭서 호출해도 무해). → **임의의 활성 InspectionSequence 인스턴스**로 호출 가능.
- 모델 경로: `ReringProject.Sequence.InspectionSequence.ResolveDatumModelPath(datum)` (static, MainView 이미 InvokeCreatePatternModel:2824 에서 사용 중).
- sequence 인스턴스 접근: `pSeq` 는 `SequenceHandler` (MainView.xaml.cs:26). 인덱서 `pSeq[ESequence.Top/Side/Bottom]` → `SequenceBase` (null 가능). `as ReringProject.Sequence.InspectionSequence` 캐스트.
</background>

```csharp
//260622 hbk Phase 57.1 Test Find 패턴 보정 연결 — TryComposeAlign 호출용 임의 활성 InspectionSequence 인스턴스 획득

private ReringProject.Sequence.InspectionSequence GetAnyInspectionSequence() {
    if (pSeq == null) return null;
    ESequence[] roles = new ESequence[] { ESequence.Top, ESequence.Side, ESequence.Bottom };
    for (int i = 0; i < roles.Length; i++) {
        ReringProject.Sequence.InspectionSequence seq = pSeq[roles[i]] as ReringProject.Sequence.InspectionSequence;
        if (seq != null) return seq;
    }
    return null;
}
```
