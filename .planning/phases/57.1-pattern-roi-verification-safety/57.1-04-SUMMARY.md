# Phase 57.1 Plan 04: Test Find 에 패턴 보정 연결 Summary

티칭 단계의 "Test Find" 버튼(`BtnTestFindDatum_Click`)을 런타임 검사와 동일한 패턴 보정 경로(`InspectionSequence.TryComposeAlign`)에 연결하여, 검사를 돌리지 않고도 Test Find 만으로 패턴매칭 → ROI 보정 + cyan 보정 ROI + `[ALIGN]/[ALIGN-ROI]` 로그를 티칭 단계에서 육안 확인할 수 있게 했다. Top/Bottom/Side 전부 동일 동작.

## What Changed

`WPF_Example/UI/ContentItem/MainView.xaml.cs` 단일 파일:

1. **`GetAnyInspectionSequence()` 헬퍼 추가** — `pSeq[ESequence.Top/Side/Bottom]` 인덱서를 순회하며 첫 번째 활성 `InspectionSequence` 인스턴스를 `as` 캐스트로 반환. `pSeq==null` 시 `null`. `TryComposeAlign` 이 sequence 실행 상태에 의존하지 않으므로(인자 입력, `_datumTransforms` 만 transient) 어느 인스턴스든 호출 가능.

2. **DualImage 분기**(`VerticalTwoHorizontalDualImage`) — 두 이미지 로드 성공 후:
   - `datum.IsPatternAlignEnabled` true → `seq.TryComposeAlign(datum, imgH, imgV, modelPath, out error)` (5-arg), `out transform` 은 `HomMat2dIdentity` 로 초기화(미사용).
   - `seq==null` → `svc.TryFindDatum(imgH, imgV, ...)` 폴백.
   - align off → 기존 `svc.TryFindDatum` 경로 유지.

3. **단일-이미지 분기**(Top/Bottom 등) — `AskTestImageSource()` 이후:
   - `datum.IsPatternAlignEnabled` true → `seq.TryComposeAlign(datum, testImage, modelPath, out error)` (4-arg).
   - `seq==null` → `svc.TryFindDatum(testImage, ...)` 폴백.
   - align off → 기존 단일-이미지 오버로드 유지.

성공 후속 처리(`SetDatumOverlay(datum, true, ...)` + `RaisePropertyChanged` + `RefreshParamEditor`)는 **무변경** — `TryComposeAlign` 이 `DetectedOrigin*`/`CurrentTransform`/`LastFindSucceeded` 를 write-back 하므로 기존 오버레이 체인이 그대로 동작.
