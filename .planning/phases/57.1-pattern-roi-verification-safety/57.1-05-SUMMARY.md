# Phase 57.1 Plan 05: 패턴 ROI 그리기 버튼 진행 확인(OK/Cancel) Summary

Datum 노드가 선택된 상태에서 "패턴 1 / 패턴 2 ROI 그리기" 버튼을 누르면 즉시 그리기 모드로 진입하던 것을, 실제 그리기 모드 진입 전에 OK/Cancel 진행 확인 메시지박스를 띄우도록 변경했다. 사용자가 [취소]하면 아무 동작 없이 종료하여, 기존 패턴 ROI 가 무심코 덮이는 것을 방지한다.

## What Changed

- `DrawPatternRoiButton_Click` (패턴 1): `datum == null` 가드 직후, `ExitCanvasMode()` 직전에 `CustomMessageBox.ShowConfirmation("패턴 1 ROI 그리기", "...새로 그리시겠습니까?...", MessageBoxButton.OKCancel) != MessageBoxResult.OK` → `return` 추가.
- `DrawPatternRoi2Button_Click` (패턴 2): 동일 위치에 패턴 2 문구로 미러 삽입.
- 두 핸들러 모두 `//260622 hbk Phase 57.1` 마커 1줄씩 동반.

## How It Works

확인 관용구는 기존 `InvokeCreatePatternModel`(:2813)의 `ShowConfirmation(... OKCancel) != MessageBoxResult.OK` early-return 패턴을 그대로 미러했다. `CustomMessageBox.ShowConfirmation(string, string, MessageBoxButton)` 은 `MessageBoxResult` 를 반환하며, OK 가 아니면 즉시 `return` 하여 `ExitCanvasMode()`/`StartRectangleDrawing()` 등 그리기 모드 진입 코드에 도달하지 않는다.

확인 분기를 `datum == null` 가드 **이후**에 배치하여, Datum 미선택 상태의 기존 알림("Datum 티칭 존을 먼저 선택하세요")은 그대로 유지된다(회귀 0). write-back 핸들러(`HalconViewer_PatternRectCompleted`/`HalconViewer_PatternRect2Completed`)와 `CreatePatternModel` 경로는 무변경.
