# 57.1-06 — Test Find 보정 ROI 박스 위치이동 표시 (UAT Test 4 fix)

<objective>
UAT 피드백: "측정(검출)은 되는데 ROI 위치이동은 하지 않아". 57.1-04 로 Test Find 가 TryComposeAlign 을 타서 검출 origin(십자)은 보정 위치로 이동하지만, **화면의 ROI 검색 박스는 공칭(티칭) 위치에 고정**돼 보정이 따라가는 게 안 보인다. 보정된 datum 검색 ROI 박스(orange)가 부품을 따라 이동해 표시되도록 한다.
</objective>

<root_cause>
- Test Find 성공 분기(MainView.xaml.cs:2991~3000)는 `halconViewer.SetDatumOverlay(datum, true, ...)` 만 호출 → Render() 의 `_datumConfig` 분기(MainResultViewerControl.xaml.cs:812~814)가 `RenderDatumOverlay` 로 **공칭 좌표** ROI 박스를 그린다(검출 십자만 DetectedOrigin 으로 이동).
- 보정된 ROI 박스(orange)는 `_resultDatumRoiOverlays` 에서 그려지며(MainResultViewerControl.xaml.cs:840~842), 이 리스트는 `MainView.ShowResultDatumOverlays(datums)`(:1302) 가 `SetResultRoiOverlays(BuildCorrectedResultRoiOverlays, BuildCorrectedDatumRoiOverlays)`(:1317) 로 채운다. `BuildCorrectedDatumRoiOverlays`(:485) 는 각 검색 ROI 를 `datum.CurrentTransform`(=TryComposeAlign 이 set 한 alignRigid) 로 변환한다.
- Test Find 는 ShowResultDatumOverlays 를 호출하지 않으므로 보정 박스가 안 그려짐 → "ROI 위치이동 안 함".
- TryComposeAlign 성공 시 `datum.CurrentTransform = alignRigid`, `datum.LastFindSucceeded = true` write-back 됨(InspectionSequence.cs:573~574) → ShowResultDatumOverlays 의 보정 빌더가 즉시 사용 가능(:1312 의 재티칭 skip 조건도 충족).
</root_cause>
