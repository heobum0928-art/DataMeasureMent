# Phase 57.1 Plan 06: Test Find 보정 ROI 박스 위치이동 표시 Summary

UAT Test 4 피드백("측정은 되는데 ROI 위치이동은 하지 않아")을 닫았다. 57.1-04 로 Test Find 가 `TryComposeAlign` 을 타서 검출 origin(십자)은 보정 위치로 이동했지만, 화면의 검색 ROI 박스는 공칭(티칭) 위치에 고정돼 보정이 따라가는 게 보이지 않았다. 보정 datum 성공 시 결과 화면과 동일한 `ShowResultDatumOverlays` 경로를 호출해, `datum.CurrentTransform` 으로 변환된 orange 검색 ROI 박스가 부품(이동/회전)을 따라 이동 표시되도록 했다.

## What Changed

`WPF_Example/UI/ContentItem/MainView.xaml.cs` 단일 파일, `BtnTestFindDatum_Click` 의 `if (ok)` 성공 분기:

기존 한 줄
```csharp
halconViewer.SetDatumOverlay(datum, true, GetDatumEditMode());
```
을 `IsPatternAlignEnabled` 게이트 분기로 교체:
- **보정(align) datum** → `halconViewer.ClearDatumOverlay()` (공칭 `_datumConfig` 박스 제거) + `ShowResultDatumOverlays(new List<DatumConfig> { datum })`. 결과 경로의 `SetResultRoiOverlays(..., BuildCorrectedDatumRoiOverlays(datums))` 가 `datum.CurrentTransform`(=`TryComposeAlign` 이 set 한 alignRigid)로 검색 ROI 박스(orange)를 변환 렌더 + 검출 십자.
- **비-align datum** → 기존 `halconViewer.SetDatumOverlay(datum, true, GetDatumEditMode())` 그대로 (공칭 ROI + 십자, 회귀 0).

`RaisePropertyChanged` / `RefreshParamEditor` 후속 처리는 **무변경**. `TryComposeAlign` 성공 시 `LastFindSucceeded=true` write-back 되므로 `ShowResultDatumOverlays`(:1312)의 재티칭 skip 조건이 충족돼 보정 좌표가 그대로 사용된다.
