# 57.1-07 — 측정 ROI 표시 90° 정상화(View) + 패턴 ROI 위치보정 표시

<objective>
UAT 피드백 2건:
- (#2) 측정 결과 화면에서 보정 측정 ROI 박스(green)가 90° 돌아 보임 → **View(표시)만의 length1/length2 스왑 버그** 수정. 측정 자체는 정상(SmallestRectangle2+measurePhi 사용).
- (#1) cyan 패턴 ROI 박스가 공칭(템플릿) 위치 고정 → datum/측정 ROI 처럼 `CurrentTransform` 으로 위치 보정된 위치에 표시.
</objective>

<root_cause>
**#2 (90° 표시 버그):**
- 측정 실제 스캔은 `FAIEdgeMeasurementService` 가 strip 영역에서 `SmallestRectangle2`(rh/rw 도출)+`GenMeasureRectangle2(rr,rc,measurePhi,rh,rw,...)`(:213/:396) 로 수행 → HALCON 내부 규약 자기일관 → **측정값 정상**.
- 표시용 `MainView.BuildCorrectedResultRoiOverlays`(:444) 는 `l1=(Row2-Row1)/2`(행/세로), `l2=(Col2-Col1)/2`(열/가로) 를 `{row,col,phi,l1,l2}` 로 `RenderResultRoiBoxes`→`DispRectangle2(...,length1=l1,length2=l2)` 에 전달(:468~475).
- HALCON `disp_rectangle2` 규약(=정상 렌더되는 cyan 패턴 ROI 와 동일): phi=0 에서 **length1 = 열(가로/X) 반장축, length2 = 행(세로/Y) 반장축**. 패턴 ROI 는 `Length1=halfW(열)`, `Length2=halfH(행)` 로 저장(HalconViewer_PatternRectCompleted:2716~2717)되어 정상 렌더됨.
- 따라서 측정 표시 박스는 length1↔length2 가 패턴/HALCON 규약과 **반대** → 비정사각형 ROI 가 90° 돌아 보임. (datum orange 박스 AppendDatumRect 는 저장된 Length1/Length2 사용 → 이 버그 없음 — 본 plan 범위 아님)

**#1 (패턴 ROI 미보정):**
- cyan 패턴 ROI 렌더(MainResultViewerControl.xaml.cs:846~858)는 `d.PatternRoi_Row/Col/Phi` 원본(공칭 템플릿 위치)만 사용 → 보정 시에도 고정.
- datum 검색 ROI 는 `BuildCorrectedDatumRoiOverlays`(MainView:485, AppendDatumRect:515) 가 `d.CurrentTransform` 으로 center 변환 + `phi + Atan2(-t[1],t[0])` 적용. 동일 방식을 패턴 ROI 에도 적용하면 매칭된(보정된) 위치로 이동.
</root_cause>
