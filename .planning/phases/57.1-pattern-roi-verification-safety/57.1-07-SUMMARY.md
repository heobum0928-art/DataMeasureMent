# Phase 57.1 Plan 07: 측정 ROI 표시 90° 정상화(View) + 패턴 ROI 위치보정 표시 Summary

ALIGN-01/02 UAT 피드백 2건을 닫았다. (#2) 측정 결과 화면의 보정 측정 ROI 박스(green)가 90° 돌아 보이던 결함은 표시용 `BuildCorrectedResultRoiOverlays` 가 length1/length2 를 HALCON `disp_rectangle2` 규약과 반대로 전달한 것이 원인이었고(측정 자체는 `FAIEdgeMeasurementService` 의 `SmallestRectangle2`+`measurePhi` 로 정상), length 계산을 규약(length1=열/가로 반장축, length2=행/세로 반장축)에 맞춰 교정해 측정값 변경 없이 표시만 정상화했다. (#1) cyan 패턴 ROI 가 공칭(템플릿) 위치에 고정돼 보정을 따라가지 않던 문제는, datum/측정 ROI 와 동일하게 `CurrentTransform` 으로 center 를 변환하고 phi 에 회전분을 가산해 매칭된 위치로 이동 표시하도록 했다.

## What Changed

### Task 1 (#2) — 측정 ROI 표시 90° 정상화 (View, commit e1bd71d)
`WPF_Example/UI/ContentItem/MainView.xaml.cs` `BuildCorrectedResultRoiOverlays`:
- `l1 = (roi.Row2-roi.Row1)/2.0; l2 = (roi.Column2-roi.Column1)/2.0` →
  `l1 = (roi.Column2-roi.Column1)/2.0` (열/가로), `l2 = (roi.Row2-roi.Row1)/2.0` (행/세로)
- HALCON `disp_rectangle2` 규약: phi=0 에서 length1=열(가로/X) 반장축, length2=행(세로/Y) 반장축 (정상 렌더되는 cyan 패턴 ROI 와 동일).
- rPhi/center 계산, `l1<=0||l2<=0` 가드, `rects.Add(...)` 무변경. 함수 상단 주석을 disp_rectangle2 규약 취지로 정정.

### Task 2 (#1) — 패턴 ROI 위치 보정 표시 (commit 088f6ab)
`WPF_Example/UI/ContentItem/MainResultViewerControl.xaml.cs` 패턴 ROI 렌더 블록:
- datum `d` 의 `CurrentTransform` 이 유효(`!= null && Length >= 5`)하면 `HOperatorSet.AffineTransPoint2d` 로 PatternRoi/PatternRoi2 center 를 변환하고 phi 에 `Math.Atan2(-pt[1].D, pt[0].D)` 회전분을 가산.
- transform 무효 시 기존 공칭 좌표 폴백 (회귀 0). length 는 disp 규약대로 이미 저장돼 변환 불필요.
- MainResultViewerControl 의 Allman brace 스타일 유지, `HTuple`/`HOperatorSet`/`Math` 모두 기존 using 으로 해결(추가 0).
