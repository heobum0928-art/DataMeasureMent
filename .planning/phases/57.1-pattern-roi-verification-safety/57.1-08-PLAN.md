# 57.1-08 — TryFitLine 에지 trim 을 "위치 정렬 + % 절사"로 개선

<objective>
UAT 피드백: 라인 피팅의 EdgeTrimCount 가 정렬 없이 누적(strip)순서 양끝만 잘라 "뒤죽박죽". 사용자 결정 = **에지점을 위치축으로 정렬 → 상위 N% + 하위 N% 제거 → 중간만 피팅**(절사 추정). EdgeTrimCount 를 양끝 각각 백분율(%)로 재해석한다.
</objective>

<root_cause>
`VisionAlgorithmService.TryFitLine`(:146~155) 의 현재 trim:
```csharp
int edgeCount = allRows.TupleLength();
if (trimCount > 0 && edgeCount > 2 * trimCount + 1) {
    HTuple trimmedR = allRows.TupleSelectRange(trimCount, edgeCount - trimCount - 1);
    HTuple trimmedC = allCols.TupleSelectRange(trimCount, edgeCount - trimCount - 1);
    allRows = trimmedR; allCols = trimmedC; edgeCount = allRows.TupleLength();
}
```
- 정렬 없음 → 누적(strip)순서 앞/뒤 trimCount 개 제거. EdgeSelection="All" 로 strip 당 다중 에지가 섞이면 위치/이상치와 무관한 점을 잘라 "뒤죽박죽".
- `edgeCount > 2*trimCount+1` 가드 → 샘플 적으면 trim 미적용(들쭉날쭉).
- 점 순서는 line fit 결과엔 무관(FitLineContourXld 는 점집합 기준) → 정렬은 오직 trim 대상 선정을 위해 필요.
</root_cause>

<action>
TryFitLine 의 trim 블록(:146~155)을 다음으로 교체한다. C# 7.2, HALCON HTuple API 사용. `//260622 hbk Phase 57.1 #2` 마커 부착. 파일 스타일(Allman) 유지.

1. **위치축 정렬 키 선택**: scanHorizontal(=LtoR/RtoL) 이면 에지가 세로로 분포 → **row 기준 정렬**, 아니면(TtoB/BtoT) → **col 기준 정렬**. (= strip 쌓이는 축)
2. `HTuple.TupleSortIndex()` 로 정렬 인덱스 획득 후 `TupleSelect(idx)` 로 allRows/allCols 동시 재정렬 (쌍 유지).
3. **양끝 % 절사**: `trimCount` 를 양끝 각각 백분율(%)로 해석. `int trimPct = Math.Min(Math.Max(trimCount, 0), 49);` 로 clamp. `removeEach = (int)(edgeCount * trimPct / 100.0);` 상위 removeEach + 하위 removeEach 제거. 단 남는 점 `>= 2` 보장(아니면 trim skip — 원본 유지).

교체 코드(가이드, 실제 HTuple 메서드 시그니처는 read_first 로 확인 후 일치):
```csharp
//260622 hbk Phase 57.1 #2 — 정렬 후 양끝 % 절사(trimmed fit). 기존: 정렬없이 누적순서 양끝 trimCount개 제거 → "뒤죽박죽".
//  위치축 정렬(scanHorizontal=세로에지→row, else→col) 후 EdgeTrimCount 를 양끝 각각 %(0~49) 로 해석해 상/하위 제거, 중간만 피팅.
//  점 순서는 FitLineContourXld 결과에 무관(점집합) — 정렬은 절사 대상 선정 목적.
int edgeCount = allRows.TupleLength();
if (trimCount > 0 && edgeCount >= 4)
{
    HTuple sortKey = scanHorizontal ? allRows : allCols;          // 분포 축
    HTuple order = sortKey.TupleSortIndex();                      // 오름차순 인덱스
    allRows = allRows.TupleSelect(order);
    allCols = allCols.TupleSelect(order);
    int trimPct = trimCount; if (trimPct > 49) trimPct = 49; if (trimPct < 0) trimPct = 0;
    int removeEach = (int)(edgeCount * trimPct / 100.0);
    if (removeEach > 0 && (edgeCount - 2 * removeEach) >= 2)
    {
        allRows = allRows.TupleSelectRange(removeEach, edgeCount - removeEach - 1);
        allCols = allCols.TupleSelectRange(removeEach, edgeCount - removeEach - 1);
        edgeCount = allRows.TupleLength();
    }
}
```
이후 `if (edgeCount < 2)` 검사(:157)·GenContourPolygonXld·FitLineContourXld·collectedEdges 누적(:163~178) 은 무변경(정렬·절사된 allRows/allCols 그대로 소비).
</action>
