# Phase 57.1 Plan 09: 에지 trim 정렬+% 절사 전 사이트 통일 (Datum 포함) Summary

57.1-08 에서 TryFitLine 에만 인라인으로 적용했던 "위치축 정렬 → 양끝 각 %(0~49) 절사" 로직을 `VisionAlgorithmService.SortAndTrimPercent(ref HTuple, ref HTuple, bool scanHorizontal, int trimPercent)` 공유 static 헬퍼로 단일 소스화하고, Datum 검출 2 사이트 + 측정 trim 2 사이트(TrimExtremePoints)를 모두 이 헬퍼에 위임하도록 교체했다. EdgeTrimCount/trimCount 는 전 사이트에서 "양끝 각 백분율(%)" 로 일관 해석된다.

## What Was Built

- **Task 1** — `VisionAlgorithmService` 에 `public static void SortAndTrimPercent(ref HTuple rows, ref HTuple cols, bool scanHorizontal, int trimPercent)` 추가. 규칙: rows/cols null 또는 trimPercent<=0 또는 점<4 → 무변경, key(scanHorizontal?row:col) TupleSortIndex 정렬 후 양끝 각 pct%(clamp 49) 제거, 남는 점<2 면 절사 skip. TryFitLine 의 57.1-08 인라인 블록(약 28줄)을 헬퍼 호출 1줄 + edgeCount 재계산으로 교체.
- **Task 2** — `DatumFindingService.TryFindLine`(~1718) 과 `TryExtractEdgePoints`(~1893) 의 개수 trim 블록(`TupleSelectRange(trimCount, edgeCount-trimCount-1)`)을 각각 `VisionAlgorithmService.SortAndTrimPercent(ref allRows, ref allCols, scanHorizontal, trimCount)` 호출로 교체. scanHorizontal 은 두 메서드 모두 기존 스코프(`direction != "TtoB" && direction != "BtoT"`) 변수를 그대로 전달. Trace 로그는 "%" 표기로 갱신.
- **Task 3** — `FAIEdgeMeasurementService.TrimExtremePoints` 와 `MeasurementAlgorithm.TrimExtremePoints` 의 자체 정렬+개수 trim 본문(각 ~33줄)을 `VisionAlgorithmService.SortAndTrimPercent(ref rows, ref cols, scanHorizontal, trimCount)` 단일 위임으로 교체. private 시그니처/호출처(254,428 / 250) 무변경.
