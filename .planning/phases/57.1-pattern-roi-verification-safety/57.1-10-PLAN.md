# 57.1-10 — EdgeTrimCount UI 를 %(비율) 표시로 통일

<objective>
trim 의미가 "양끝 각 백분율(%)" 로 바뀌었으므로(57.1-08/09), PropertyGrid 라벨을 % 로 표시한다. **프로퍼티명/INI 키는 보존**(하위호환 유지)하고 `[DisplayName]` 어노테이션만 추가한다.
</objective>

<background>
- PropertyTools PropertyGrid 는 [DisplayName] 있으면 그걸, 없으면 C# 프로퍼티명을 라벨로 표시.
- 현재 trim 필드들은 [DisplayName] 미사용 → "EdgeTrimCount" 그대로 노출.
- INI/JSON 직렬화(ParamBase reflection / Newtonsoft)는 **프로퍼티명** 기준 → [DisplayName] 은 직렬화에 무영향(호환 안전).
- DisplayName 어노테이션 = `PropertyTools.DataAnnotations.DisplayName` (이 프로젝트가 PropertyTools.DataAnnotations 를 사용 — 기존 [Category]/[ItemsSourceProperty] 와 동일 namespace 확인 후 사용).
</background>

<tasks>
<task id="1" type="execute">
<name>측정 클래스 13개의 EdgeTrimCount 에 [DisplayName("Edge Trim (%)")] 추가</name>

<read_first>
- WPF_Example/Custom/Sequence/Inspection/Measurements/EdgeToLineAngleMeasurement.cs (EdgeTrimCount:29, 기존 어노테이션 패턴 [Category]/[ItemsSourceProperty] 의 DisplayName namespace 확인)
- WPF_Example/Custom/Sequence/Inspection/Measurements/EdgeToLineDistanceMeasurement.cs
- WPF_Example/Custom/Sequence/Inspection/Measurements/DualImageEdgeDistanceMeasurement.cs (PointROI_EdgeTrimCount + LineROI_EdgeTrimCount 2개)
- (나머지 측정 파일들은 EdgeTrimCount 선언 라인만 확인)
</read_first>

<action>
아래 측정 클래스 각 파일에서 `public int EdgeTrimCount { get; set; }` 선언 **바로 위**에 `[DisplayName("Edge Trim (%)")]` 추가. DualImage 는 PointROI_EdgeTrimCount → `[DisplayName("Point Edge Trim (%)")]`, LineROI_EdgeTrimCount → `[DisplayName("Line Edge Trim (%)")]`. `//260622 hbk Phase 57.1` 주석. 프로퍼티명/기본값/타입 무변경.

대상(EdgeTrimCount 보유):
ArcEdgeDistanceMeasurement, CompoundAngleMeasurement, CompoundCenterBDistanceMeasurement, CompoundCenterCDistanceMeasurement, DualImageEdgeDistanceMeasurement(2개), EdgePairDistanceMeasurement, EdgeToLineAngleMeasurement, EdgeToLineDistanceMeasurement, LineToLineAngleMeasurement, LineToLineDistanceMeasurement, PointToLineDistanceMeasurement, PointToPointDistanceMeasurement.

DisplayName 어노테이션 namespace 는 기존 [Category] 와 동일하게 `PropertyTools.DataAnnotations.DisplayName` 사용 (using PropertyTools.DataAnnotations 이미 있으면 `[DisplayName(...)]`, 충돌 시 완전한정명). 일부 파일에 EdgeTrimCount 가 없으면(CircleCenter/CircleDiameter/CompoundShortAxis/ArcLineIntersect 등 조사상 미보유) skip — frontmatter files_modified 에 없는 파일은 건드리지 않음.
</action>

<acceptance_criteria>
- 13개 측정 클래스(DualImage 2필드 포함)의 EdgeTrimCount 선언 위에 [DisplayName("...(%)")] 존재: `grep -rl "DisplayName(\"Edge Trim (%)\")\|DisplayName(\".*Edge Trim (%)\")" Measurements/` 가 대상 파일 커버
- 프로퍼티명 EdgeTrimCount / PointROI_EdgeTrimCount / LineROI_EdgeTrimCount 무변경 (INI 호환)
- MSBuild 0 errors
</acceptance_criteria>
</task>
