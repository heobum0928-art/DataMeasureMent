# Phase 57.1 Plan 10: EdgeTrimCount UI 를 %(비율) 표시로 통일 Summary

trim 의미가 "양끝 각 백분율(%)"(57.1-08/09)로 바뀜에 따라, 19개 trim 프로퍼티(측정 13 + Datum 6)에 PropertyTools `[DisplayName("...Edge Trim (%)")]` 어노테이션만 추가하여 PropertyGrid 라벨을 % 표기로 통일했다. 프로퍼티명/INI 키/기본값/타입을 모두 보존해 기존 레시피 직렬화 호환은 완전 무영향이다.

## What Was Built

### Task 1 — 측정 클래스 13개 trim 필드 라벨 (commit a2e6f10)
- 12개 측정 클래스의 `EdgeTrimCount` 위에 `[DisplayName("Edge Trim (%)")]` 추가:
  ArcEdgeDistance, CompoundAngle, CompoundCenterBDistance, CompoundCenterCDistance,
  EdgePairDistance, EdgeToLineAngle, EdgeToLineDistance, LineToLineAngle,
  LineToLineDistance, PointToLineDistance, PointToPointDistance.
- DualImageEdgeDistance 2필드:
  `PointROI_EdgeTrimCount` → `[DisplayName("Point Edge Trim (%)")]`,
  `LineROI_EdgeTrimCount` → `[DisplayName("Line Edge Trim (%)")]`.
- 모든 파일에 `//260622 hbk Phase 57.1` 주석. `using PropertyTools.DataAnnotations;` 가 전 파일에 이미 존재해 `[DisplayName(...)]` 직접 사용.

### Task 2 — DatumConfig 6개 trim 필드 라벨 (commit a1a167d)
- `Line1_/Line2_/Circle_/Vertical_/Horizontal_A_/Horizontal_B_ EdgeTrimCount` 위에 각각 `[DisplayName("Edge Trim (%)")]` 추가.
- 기존 컬럼 정렬 스타일(`public int    Name { get; set; } = 0;`) 보존.
- `EnsurePerRoiDefaults` 기본값 로직 무수정.

## Key Design / Compatibility
- PropertyTools PropertyGrid 는 `[DisplayName]` 존재 시 라벨로 사용, 미존재 시 프로퍼티명 사용.
- ParamBase reflection / Newtonsoft 직렬화는 **프로퍼티명** 기준 → `[DisplayName]` 추가는 INI/JSON 키에 무영향. 기존 레시피 로드/저장 회귀 0.
- 값 해석(% 절사)은 57.1-09 의 `VisionAlgorithmService.SortAndTrimPercent` 공유 헬퍼가 담당. 본 plan 은 표시 라벨만 변경.
