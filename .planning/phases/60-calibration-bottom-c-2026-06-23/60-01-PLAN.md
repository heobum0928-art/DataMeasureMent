NOTE on reflection Load behavior (reference_parambase_missing_key_zeroes_default.md):
ParamBase.Load auto-serializes public properties by type; a missing INI key is
LOADED AS 0 for double. Because the picker-center default IS 0 (= uncalibrated),
NO AfterLoad restore is strictly required to avoid data loss — but per Phase 58
pattern we still add an explicit no-op-style guard method so the intent
("0 = uncalibrated, leave as-is") is documented and future non-zero defaults are
easy to add. Do NOT force a non-zero default — 0 is the correct uncalibrated value.
