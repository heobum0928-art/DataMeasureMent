**왜 필요:** Phase 61 실측에서 ① Grab 이 `D:\align_test.bmp` 한 장만 폴백 로드 → 이동/회전 등 다양한 케이스로 offset/theta 검증 불가(사용자 fail 지목), ② 매칭 결과가 좌측 텍스트(X/Y/Theta/Score)로만 표시되고 이미지 위에 검출 위치/ROI/에지가 안 보임.

**범위 밖:** 피커캘 실측(36스텝 회전 = 실 피커 HW 필요, 별개). 신규 매칭 알고리즘. Grabber 검사 경로.

<decisions>
## Implementation Decisions

### D-01: Offline 이미지 로더 = 폴더 + 이전/다음
- Tray/Bottom 뷰(`TrayVisionView`/`BottomVisionView`) 툴바에 **[폴더 열기]** + **[◀ 이전] [다음 ▶]** 버튼 추가.
- [폴더 열기] → 폴더 선택 → 폴더 내 이미지 파일 목록(bmp/png/jpg/tif) 로드 → 인덱스 0. [이전]/[다음]으로 순회하며 연속 테스트(예: 16세트).
- **로드한 이미지가 "현재 이미지"가 되어** 티칭/Run/캘의 입력으로 사용됨(`MainResultViewerControl.LoadImage` + 뷰 내부 보관 `HImage`). 즉 Grab 없이 임의 이미지로 매칭 검증.
- **기존 Grab/Live/Stop + `D:\align_test.bmp` 폴백 동작은 그대로 유지**(회귀 0). 로더는 추가 경로.
- 폴더 선택 다이얼로그는 기존 의존성 `Ookii.Dialogs.Wpf` 사용 권장.

### D-02: 결과 시각화 범위 = 검출 + 보정 ROI + 에지 전부 (사용자 확정)
- Align Run 후 이미지 위에 표시: (a) **검출 위치**(pose 중심 십자), (b) **보정(offset/theta)으로 이동한 ROI 박스**, (c) **검출 에지**(matched model contour).
- 이를 위해 **`AlignShapeMatchService` 확장**(anti-goal 일부 해제) — `Run` 이 검출 pose(패턴별 row/col/angle) + 보정 ROI 박스 좌표 + 검출 에지 contour 를 추가 반환. **기존 `Run(img, mode)` 호출 + `AlignResult` 필드(offset/score/theta/found)는 하위호환 유지**(out 파라미터 추가 또는 AlignResult 필드 확장 — planner 결정).

### D-03: View on/off 토글 + 표시 = MainResultViewerControl 기존 API 재사용
- 표시는 신규 렌더러 만들지 말고 **공유 `MainResultViewerControl` 기존 오버레이 API 호출**:
  - 토글: `SetMeasurementOverlayVisible` / `SetDatumOverlayVisible` / `SetPatternRoiOverlayVisible`
  - 보정 ROI 박스: `SetResultRoiOverlays`
  - 에지: `SetInspectionOverlays` / `UpdateDisplayState`
  - 검출 십자: `SetDatumFindResultOverlay` 등 기존 십자 오버레이 활용 가능
- align 뷰 좌측 패널에 **체크박스**(ROI 표시 / 에지 표시) 추가 → 위 토글 호출. airspace 준수(체크박스는 뷰어 호스트 밖, 오버레이는 HALCON 창 내부 렌더) [[feedback_halcon_hwnd_airspace]].

### D-04: anti-goal (이번엔 AlignShapeMatchService만 해제)
- **수정 허용:** `AlignShapeMatchService`(시각화 데이터 반환 확장), `TrayVisionView`/`BottomVisionView`(로더 버튼·시각화 체크박스·배선), csproj(필요 시).
- **여전히 무수정:** `MainView`/`MainView.xaml.cs`, `MainResultViewerControl`(consume only — 기존 public API 호출), `EthernetAlignCamera`/`PickerCenterCalibrationService`, Grabber(Sequence/Device/SystemHandler), 검사 경로.
- `MainResultViewerControl` 오버레이 API가 align 결과 형태와 안 맞으면, 뷰 레벨에서 데이터를 기존 API 시그니처에 맞춰 변환(서비스 수정으로 흡수). 그래도 부족하면 planner가 최소 확장 제안 → 재논의.

### D-05: 피커캘 실측은 범위 밖
- 피커센터 캘(60-1/2) 실값 검증은 실 피커 36스텝 회전 HW 필요 → Phase 61.1 범위 아님. HW 합류 시점에 별도 진행.

### Claude's Discretion
- 로드 폴더 마지막 위치 기억 여부, Tray/Bottom 공통 로더 vs 각 뷰 독립, 지원 이미지 확장자 집합, 시각화 색/선 두께/십자 크기, 체크박스 레이아웃 위치, AlignShapeMatchService 확장 방식(out 파라미터 vs AlignResult 필드 추가), 에지 contour 단순화 수준.
</decisions>
