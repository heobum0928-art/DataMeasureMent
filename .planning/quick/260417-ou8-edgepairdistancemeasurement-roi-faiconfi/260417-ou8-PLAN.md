# Plan: FAIConfig ROI 단일 소스화

## 문제

`FAIConfig.ROI_*`(표시용)와 `EdgePairDistanceMeasurement.ROI_*`(측정용)가 중복 저장.
`CommitRectRoi`의 동기화 블록이 DataGrid 선택 조건에 걸려 자주 스킵 → 노란 ROI(사용자 입력) ≠ 빨간 ROI(측정이 실제 사용하는 좌표).

## 해결 전략

중복 저장 구조 자체를 제거. EdgePair는 Owner(FAIConfig)를 단일 소스로 참조.

## Tasks

### T1. EdgePairDistanceMeasurement.cs — ROI 필드 제거 + Owner 참조
- `ROI_Row/Col/Phi/Length1/Length2` 5개 프로퍼티 삭제
- `TryExecute` 상단에서 `Owner as FAIConfig` 캐스팅, null이면 error 반환
- 임시 FAIConfig 구성 시 ROI_*는 `ownerFai.ROI_*`에서 복사
- Edge/Calibration 파라미터는 self 유지

### T2. MainView.xaml.cs `CommitRectRoi` — Measurement 동기화 블록 제거
- 라인 571-580 (SelectedItem.SourceMeasurement 캐스팅 + edgeMeas.ROI_* 대입) 삭제
- 삭제 이유 주석 한 줄 남김

## Verification
- 전체 프로젝트에서 `edgeMeas.ROI_` / `EdgePairDistanceMeasurement.*ROI_` 참조 없음
- 런타임: FAI_0 ROI 새로 그린 후 측정 실행 → 측정 ROI가 사용자 ROI와 일치

## 하위호환
레거시 INI에 `EdgePair.ROI_*` 키가 남아있어도 `ParamBase.Load` 리플렉션이 미존재 프로퍼티를 무시하므로 로드 실패 없음.
