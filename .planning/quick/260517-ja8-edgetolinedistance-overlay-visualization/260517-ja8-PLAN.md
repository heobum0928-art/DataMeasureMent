1. **TryFitLine 의 pr1/pc1/pr2/pc2 는 IMAGE 좌표다.**
   VisionAlgorithmService.TryFitLine (VisionAlgorithmService.cs:41-58) 은 datumTransform 으로 ROI 좌표를
   image 좌표(rRow/rCol/rPhi)로 *변환*한 뒤 그 위치에서 MeasurePos 를 실행한다. FitLineContourXld
   (L168-172) 가 반환하는 lr1/lc1/lr2/lc2 = pr1/pc1/pr2/pc2 는 image 공간 위의 피팅 라인 endpoint 다.
   → 검출 에지 라인 overlay 는 pr1/pc1/pr2/pc2 를 **그대로** 사용한다 (역변환 불필요).
   → 에지 중점 (pRow, pCol) = ((pr1+pr2)/2, (pc1+pc2)/2) 도 image 좌표다 (TryExecute.cs:95-96 에 이미 계산됨).

2. **교점(Datum 원점)의 image 좌표 = datumTransform 의 역행렬을 (0,0) 에 적용한 결과다.**
   datumTransform 은 image 좌표 → datum-relative 좌표 변환 행렬이다 (TryExecute.cs:103 이 측정점에
   AffineTransPoint2d 로 이 변환을 적용함). 따라서 datum 원점(datum-relative 의 0,0)의 image 좌표는:
     HomMat2dInvert(datumTransform) → invMat
     AffineTransPoint2d(invMat, 0, 0) → (originRow, originCol)
   DatumConfig.DetectedOriginRow/Col 은 채택하지 않는다 — 이 필드는 DatumFindingService.TryFindDatum
   (find 경로) 에서만 write-back 되는 transient 이며, TryExecute 는 DatumConfig 참조 없이 datumTransform
   만 받으므로 populate 보장이 없다. datumTransform 역변환이 self-contained 하고 결정적이다.
