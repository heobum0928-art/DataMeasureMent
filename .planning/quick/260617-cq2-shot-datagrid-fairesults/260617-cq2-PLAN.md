# Quick Task 260617-cq2: 일괄 검사 결과 그리드 표시

## Problem

일괄 검사(`Btn_batchRun_Click` → `BatchRunService.StartBatch`)는 완료 시
`_batchAccumulated` 에 DTO 만 누적해 xlsx Export 용으로 쓰고, 결과 그리드
`dataGrid_faiResults` 를 전혀 건드리지 않는다. 이 그리드는
`InspectionViewModel.MeasurementResults` 에 바인딩되어 **현재 선택된 단일 노드**
(FAI/Shot)의 측정만 행으로 빌드하고 검사 후 `row.Refresh()` 로 값만 갱신한다.

→ 체크박스로 여러 SHOT 을 일괄 검사해도 리스트박스에 결과가 표시되지 않는다.
(Phase 51 설계 갭 — 라이브 그리드 표시는 범위 밖, Export 로만 검증되었음)

## Fix (전체 펼침 — 검사한 체크 SHOT 전부 표시)

행은 live `MeasurementBase` 를 감싸므로 검사 후 `LastMeasuredValue` 가 이미 채워져 있다.
`FAIName` 은 순수 유지 — MainView 가 동일-명 FAI 를 `SourceMeasurement` ReferenceEquals
(`MainView.xaml.cs:292`)로 구분하므로 ROI 하이라이트/편집 회귀 0.

### Task 1 — InspectionViewModel 에 다중 SHOT 표시 메서드 추가
- files: `WPF_Example/UI/ViewModel/InspectionViewModel.cs`
- action: `ShowMeasurementsForShots(List<ShotConfig> shots)` 추가 —
  주어진 shots 의 모든 FAI × Measurement 를 평탄화해 `MeasurementResultRow` 행으로
  `MeasurementResults` 교체. null/빈 입력 방어. `OnActionSelected` 의 단일-shot 로직을
  다중 shot 으로 확장한 형태(코드 패턴 동일).
- verify: 메서드가 `List<ShotConfig>` 를 받아 `fai.FAIName` 순수 유지로 행 생성
- done: 빌드 0 errors

### Task 2 — 일괄 검사 완료 시 그리드 채우기
- files: `WPF_Example/UI/ControlItem/InspectionListView.xaml.cs`
- action:
  1. 필드 `private List<ShotConfig> _batchShots;` 추가
  2. `Btn_batchRun_Click` 에서 체크된 ShotConfig 목록을 `_batchShots` 에 저장
     (이미 순회하는 `checkedShots` 의 `n.Param as ShotConfig` 재사용)
  3. `OnBatchComplete` Dispatcher 블록에서 누적 후
     `_inspectionVm.ShowMeasurementsForShots(_batchShots)` 호출
- verify: 일괄 검사 완료 → 그리드에 체크 SHOT 전체 측정 행 + 측정값/판정 표시
- done: 빌드 0 errors
