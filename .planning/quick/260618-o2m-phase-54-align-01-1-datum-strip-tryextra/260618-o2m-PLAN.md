# Quick Task 260618-o2m: datum 검출 strip θ 회전

## 문제 (Phase 54 ALIGN-01 carry-over #1)

패턴매칭 위치보정(ALIGN)에서 **측정 ROI는** 보정 transform 의 회전각까지 적용해 strip 을
자재 틸트만큼 기울이는데(`FAIEdgeMeasurementService.cs:63/100/111`), **datum 검출 strip은**
중심 이동만 적용하고 회전을 빠뜨림(`DatumFindingService.TryExtractEdgePoints`). 축정렬 strip 이
틸트된 datum 에지를 비스듬히 샘플 → `DetectedRefAngle` ~0.1-0.2° 편차 → datum 에서 먼
측정점(A1/A2/A3) NG, 가까운 것(A4~6) OK (지렛대 효과 = 각도 오차의 지문).

## 수정 (측정 ROI 서비스 미러링)

`DatumFindingService.cs`:
1. `TryExtractEdgePoints`: `AlignPreTransform` 에서 회전각 추출
   `alignRot = Atan2(T[3], T[0])` (CANONICAL: FAIEdgeMeasurementService.cs:63).
2. bounding box 를 회전 반영 enlarged AABB 로 교체. **datum 규약 length1=col / length2=row**
   (측정 서비스와 반대) 보존: `halfCol=|L1·cosθ|+|L2·sinθ|`, `halfRow=|L1·sinθ|+|L2·cosθ|`.
   `alignRot=0` → `halfCol=L1, halfRow=L2` 로 기존 축정렬 동작 정확 복원(회귀 0).
3. `AppendEdgePointsFromStrip` 에 선택적 파라미터 `double alignRot = 0.0` 추가 →
   `measurePhi += alignRot`. **TryFindLine(비-align 경로) 2개 호출은 기본값 0 으로 무변경.**

`InspectionSequence.cs` `TryComposeAlign`:
4. 확증 로그 추가 — datum 검출각(`DetectedAngleDeg`) 회전분 vs 패턴 θ.
   수정 후 둘 수렴(편차~0) = 메커니즘 확증.
