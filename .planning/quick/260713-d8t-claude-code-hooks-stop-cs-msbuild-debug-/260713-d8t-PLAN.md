**stop-build-verify.js (Stop, BLOCKING build gate):**
1. Parse stdin. If `data.stop_hook_active === true` -> `process.exit(0)` (documented infinite-loop guard).
2. Gate: run `git status --porcelain -- '*.cs'` under project root (`process.env.CLAUDE_PROJECT_DIR` or `C:/Info/Project/DataMeasurement`) via `child_process.execFileSync('git', [...], {cwd})`. If output is empty (no .cs changes) -> `process.exit(0)`. Do not build every turn.
3. If .cs changes exist: run MSBuild via `spawnSync` (or `execFileSync`) with `timeout: 60000`:
   - exe: `C:/Program Files/Microsoft Visual Studio/2022/Community/MSBuild/Current/Bin/MSBuild.exe`
   - args: `["C:/Info/Project/DataMeasurement/WPF_Example/DatumMeasurement.csproj", "-p:Configuration=Debug", "-p:Platform=x64", "-nologo", "-v:m", "-clp:ErrorsOnly"]`
   - Do NOT interpolate any session/stdin data into the command. Args are a static array.
4. If MSBuild exe path does not exist (`fs.existsSync` false), OR spawn throws, OR timed out (`result.error` / `result.status === null` from timeout) -> write a note to `process.stderr`, `process.exit(0)` (FAIL OPEN — never brick session on env drift).
5. Combine stdout+stderr of the build. If it contains `error CS` -> collect the first ~10 lines matching `error CS`, build `reason` string, write `{"decision":"block","reason":<reason>}` to stdout via `console.log(JSON.stringify(...))`, `process.exit(0)`.
6. Else (build clean) -> `process.exit(0)` with no output (allow stop).

**orphan-cs-detect.js (PostToolUse Write, non-blocking warn):**
1. Parse stdin. `filePath = data.tool_input?.file_path`. If not a string ending in `.cs` (case-insensitive) -> `process.exit(0)`.
2. Skip dated backup files: if the base filename (before `.cs`) ends with a `_MMDD`-style date suffix (regex e.g. `/_\d{3,4}$/` on the stem, e.g. `Action_BottomInspection_0428`) -> `process.exit(0)` (CLAUDE.md: dated backups are not compiled).
3. Read `WPF_Example/DatumMeasurement.csproj`. Extract every `Include="..."` value from `<Compile Include="...">` elements (regex `/<Compile\s+Include="([^"]+)"/g`). Normalize each: lowercase, backslash->forward-slash.
4. Compute the written file's path relative to `WPF_Example/`. Take `filePath`, normalize slashes+lowercase, and derive the portion after `wpf_example/`. Compare against the normalized Include set.
5. If NOT found in the Include set -> emit non-blocking warning via stdout JSON `hookSpecificOutput.additionalContext`: tell Claude the file must be added to `<Compile Include>` in DatumMeasurement.csproj to actually compile. Always `process.exit(0)`.

**cs-style-warn.js (PostToolUse Edit|Write, non-blocking, ADDED-LINE ONLY):**
1. Parse stdin. `filePath = data.tool_input?.file_path`. If not ending `.cs` -> `process.exit(0)`.
2. Determine scanned text: if `data.tool_name === 'Write'` -> scan `data.tool_input.content`. If `data.tool_name === 'Edit'` -> scan ONLY `data.tool_input.new_string` (NEVER whole file — legacy code has 63 ternaries etc. that would false-positive). If neither / empty -> `process.exit(0)`.
3. Check A (C# 8+ pinned-to-7.2 per CLAUDE.md): match against scanned text:
   - `record` type decl: `/\brecord\s+\w+/`
   - null-coalescing assignment: `/\?\?=/`
   - `using var`: `/\busing\s+var\b/`
   Collect any hit with a short label + the CLAUDE.md "C# 7.2 pinned" reason.
4. Check B (HALCON SetColor invalid color -> silent render failure, per past incident):
   - Find `/SetColor\s*\([^)]*"([^"]+)"\s*\)/g` captures (last string arg).
   - Whitelist (case-insensitive): red, green, blue, cyan, magenta, yellow, white, black, orange, gray, grey, "slate blue", brown, pink, violet, "light gray", "dark gray", "forest green", "gold", "coral". Also accept hex matching `/^#[0-9A-Fa-f]{6}$/`. Any captured literal not whitelisted and not hex -> collect with the "non-standard HALCON color -> exception swallowed -> silent no-render" reason.
5. If any warnings collected -> emit stdout JSON `hookSpecificOutput.additionalContext` with a short bullet list. Always `process.exit(0)`.
