파일 전체를 먼저 읽어 정확한 현재 줄번호를 확인한 뒤 아래 5가지 변경을 정확히 적용한다. Allman 브레이스, C# 7.2/.NET Framework 4.8 규약, 이 파일의 기존 주석 밀도를 따른다. 다른 property/method 는 절대 건드리지 않는다.

1. `MeasCorrectionFactor` 바로 위(같은 Measurement|Tolerance 그룹)에 새 public property `MeasCorrectionEnabled` (bool, 기본값 `false`) 추가. `[Category("Measurement|Tolerance")]` + 짧은 `[System.ComponentModel.Description(...)]` — 아래 보정계수의 opt-in 게이트임을 설명(예: "측정별 보정계수 사용 여부. 켜야 아래 MeasCorrectionFactor 가 적용된다. 기본 꺼짐(무보정)."). 이 새 bool 에는 커스텀 Load() override 를 추가하지 않는다 — ParamBase 의 generic reflection Boolean 경로가 키 부재를 IniValue.Default.ToBool()=false 로 처리하므로 "기본 opt-in off" 안전 동작이 이미 보장됨 (MeasCorrectionFactor 의 위험한 0-default 케이스와 정반대 리스크 프로파일).

2. `MeasCorrectionFactor` 에서 `[PropertyTools.DataAnnotations.Browsable(false)]` 한 줄만 제거하여 PropertyGrid 에 다시 노출. `[Category]`, `[Description]`, 기본값 1.0, getter/setter 는 그대로 둔다.

3. `EvaluateJudgement` 의 보정 게이트를 `if (MeasCorrectionEnabled && AppliesCorrectionFactor && MeasCorrectionFactor > 0.0)` 로 변경 — 순수 추가 AND 조건. 이 메서드의 그 외 로직, InvertSign/UseAbsoluteValue 대비 실행 순서, 다른 어떤 줄도 변경하지 않는다. OR/부호반전 금지.

4. `MeasCorrectionFactor` 위 기존 설명 주석 블록을 새 현실에 맞게 갱신: 더 이상 숨김이 아니며, 보정은 Shot-only 가 아니라 `MeasCorrectionEnabled` 를 통한 opt-in 임을 반영. 아직 유효한 경고는 보존 — (a) 반복성 요구: 여러 부품의 vision↔현미경 상관에서 도출한 값만 사용, 단일 샘플 금지, (b) 실제 부품 산포/tilt/안착 문제를 보정으로 은폐 금지, (c) 각도 타입은 보정 미적용.

5. `AppliesCorrectionFactor` 의 기존 타입별 override 동작은 절대 건드리지 않는다 — `MeasCorrectionEnabled` 는 그 위에 얹는 직교(orthogonal) per-INSTANCE opt-in 레이어이지 대체가 아니다. `Load(IniFile, string)` override(MeasCorrectionFactor 1.0 복원)도 불변 유지.
